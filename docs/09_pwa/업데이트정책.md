## 1. 문서 목적

본 문서는 미냥이 지갑 PWA의 버전 관리, 업데이트 감지, 캐시 갱신 및 사용자 업데이트 처리 기준을 정의한다.

PWA는 브라우저 캐시와 Service Worker를 사용하므로 일반 웹 서비스와 다른 업데이트 정책이 필요하다.

# 2. 업데이트 정책 개요

미냥이 지갑은 Service Worker 기반 버전 관리를 사용한다.

새 버전 배포 시 기존 사용자는 즉시 새 버전을 사용하지 않을 수 있다.

따라서 다음 과정을 통해 업데이트를 수행한다.

```text
새 버전 배포
 ↓
새 Service Worker 설치
 ↓
업데이트 감지
 ↓
사용자 안내
 ↓
업데이트 적용
```

---

# 3. 업데이트 원칙

## 원칙 1

사용자 데이터가 손실되면 안 된다.

---

## 원칙 2

동기화 대기 데이터가 존재할 경우 강제 업데이트하지 않는다.

---

## 원칙 3

업데이트는 사용자가 인지할 수 있어야 한다.

---

## 원칙 4

캐시는 버전 단위로 관리한다.

---

## 원칙 5

구버전 캐시는 자동 정리한다.

---

# 4. 버전 관리 정책

## 버전 구성

```text
MAJOR.MINOR.PATCH
```

예시

```text
1.0.0
1.0.1
1.1.0
2.0.0
```

---

## 버전 의미

### MAJOR

구조 변경

예시

```text
DB 구조 변경
동기화 구조 변경
```

---

### MINOR

기능 추가

예시

```text
새 화면 추가
새 통계 기능 추가
```

---

### PATCH

버그 수정

예시

```text
UI 수정
오류 수정
```

# 5. 업데이트 감지

Service Worker는 새 버전을 감지한다.

```text
기존 Service Worker
 ↓
새 버전 확인
 ↓
update_available
```

---

상태 변경

```text
online
 ↓
update_available
```

# 6. 업데이트 상태 정의

| 상태               | 설명                                   |
| ---------------- | -------------------------------------- |
| update_available | 업데이트 가능                              |
| update_blocked   | 미동기화 데이터 존재로 업데이트 적용 보류 중            |
| updating         | 업데이트 적용 중                            |
| update_completed | 업데이트 완료                              |
| update_failed    | 업데이트 실패                              |

> `update_blocked`는 `update_available` 상태에서 `pending_sync` 또는 `syncing` 상태가 함께 존재할 때 전환된다.
> 동기화 완료(`synced`) 후 자동으로 `update_available`로 재전환되어 업데이트를 진행한다.
> 앱 전체 우선순위 기준은 `docs/09_pwa/PWA상태정의.md`를 따른다.

---

상태 흐름

```text
update_available
 ↓ (pending_sync 존재 시)
update_blocked
 ↓ (동기화 완료 후)
update_available
 ↓
updating
 ↓
update_completed
```

---

즉시 적용 흐름

```text
update_available
 ↓ (Sync Queue 없음)
updating
 ↓
update_completed
```

---

실패

```text
update_available
 ↓
updating
 ↓
update_failed
```

---

# 20. 최종 목표

미냥이 지갑의 업데이트 기능은 새로운 기능을 빠르게 제공하면서도 사용자 데이터의 안정성을 보장해야 한다.

최종적으로 다음을 만족해야 한다.

* 자동 버전 감지
* 사용자 안내 제공
* 캐시 버전 관리
* Sync Queue 보호
* 데이터 손실 방지
* 안전한 업데이트 적용
* 실패 시 복구 가능
* 오프라인 환경 고려
