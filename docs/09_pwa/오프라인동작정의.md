# 오프라인동작정의

## 1. 문서 목적

본 문서는 미냥이 지갑이 오프라인 상태에서 어떻게 동작하는지 정의한다.

오프라인 지원의 목적은 네트워크가 불안정하거나 연결이 끊어진 상황에서도 사용자가 핵심 기능을 계속 사용할 수 있도록 하는 것이다.

본 문서는 다음 문서와 연계된다.

* PWA개요
* ServiceWorker정의
* 캐시정책
* 동기화구조
* PWA상태정의
* 예외처리
* 오프라인정책

---

# 2. 오프라인 지원 원칙

미냥이 지갑은 다음 원칙을 따른다.

### 원칙 1

조회 기능은 가능한 범위 내에서 지원한다.

### 원칙 2

거래 입력은 오프라인 저장을 지원한다.

### 원칙 3

서버 검증이 필요한 기능은 제한한다.

### 원칙 4

데이터 손실이 발생하지 않아야 한다.

### 원칙 5

네트워크 복구 후 자동 동기화를 수행한다.

# 18. 동기화 대기 상태

오프라인 저장 거래는

```text
pending_sync
```

상태를 가진다.

계좌 거래는 오프라인 저장 시 로컬 Queue와 캐시된 계좌/거래 데이터를 기준으로 날짜별 잔액 흐름을 1차 검증한다. 서버 동기화 시 서버가 동일한 정책으로 최종 검증하며, `ACCOUNT_004`가 반환되면 해당 Queue 항목은 동기화 실패 상태로 유지하고 사용자가 금액, 날짜, 계좌를 수정할 수 있게 안내한다.

---

상태 흐름

```text
pending_sync
 ↓
syncing
 ↓
synced
```

---

실패

```text
pending_sync
 ↓
sync_failed
```

# 20. 제한 기능 정의

오프라인 상태에서는 다음 기능을 지원하지 않는다.

| 기능       | 사유        |
| -------- | --------- |
| 로그인      | 인증 서버 필요  |
| 회원가입     | 서버 저장 필요  |
| 비밀번호 재설정 | 이메일 인증 필요 |
| 계좌 등록    | 정합성 검증 필요 |
| 카드 등록    | 정합성 검증 필요 |
| 카테고리 등록  | 정합성 검증 필요 |
| 아이콘 등록   | 서버 저장 필요  |

# 24. 최종 목표

미냥이 지갑은 네트워크 연결이 없는 환경에서도 핵심 기능을 사용할 수 있어야 한다.

최종적으로 다음을 만족해야 한다.

* 대시보드 조회 가능
* 거래 등록 가능
* 거래 수정 가능
* 거래 삭제 가능
* 통계 조회 가능
* 자동 동기화 가능
* 데이터 손실 방지
* 사용자 상태 명확히 표시
