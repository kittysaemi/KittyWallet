# 5. 오프라인 수정/삭제 정책

오프라인 상태에서도 거래 등록, 수정, 삭제를 지원한다.

단, 서버 반영은 네트워크 복구 후 Sync Queue를 통해 수행한다.

| 상황 | 처리 |
|---|---|
| 신규 거래 등록 | CREATE Queue 등록 |
| 서버 반영 전 거래 수정 | 기존 CREATE Queue의 payload 갱신 |
| 서버 반영 완료 거래 수정 | UPDATE Queue 등록 |
| 서버 반영 전 거래 삭제 | CREATE Queue 제거 및 로컬 데이터 삭제 |
| 서버 반영 완료 거래 삭제 | DELETE Queue 등록 및 로컬 deleted_yn 반영 |

삭제 우선순위:

1. 동일 거래에 DELETE Queue가 존재하면 UPDATE Queue를 추가하지 않는다.
2. 미동기화 CREATE 거래를 삭제하면 서버 호출 대상이 아니므로 Queue를 제거한다.
3. 서버 반영 완료 거래 삭제는 Soft Delete 기준으로 동기화한다.

# 6. 로컬 데이터 정리 정책

| 상황      | 처리                                    |
| ------- | ------------------------------------- |
| 로그아웃    | IndexedDB 유지 (재로그인 시 동기화 재개)          |
| 회원탈퇴    | IndexedDB 전체 삭제 (deleteDatabase)      |

회원탈퇴 시 서버 데이터 물리 삭제 정책과 동일하게 로컬 저장 데이터도 전체 삭제한다.
