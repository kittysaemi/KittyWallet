# 로그인정책

## 1. 문서 목적

본 문서는 사용자 인증 및 로그인 처리 기준을 정의한다.

---

# 2. 인증 방식

| 항목     | 정책                              |
| ------ | ------------------------------- |
| 인증 방식  | JWT 기반 인증 사용                    |
| 로그인 방식 | 이메일 기반 로그인                      |
| 인증 유지  | Access Token + Refresh Token 사용 |
| 사용자 식별 | user_id 기준 처리                   |

| 항목      | 정책              |
| ------- | --------------- |
| 이메일 중복  | 동일 이메일 중복 가입 불가 |

# 4. 로그인 정책

| 항목        | 정책                  |
| --------- | ------------------- |
| 로그인 실패    | 오류 메시지 표시           |
| 로그인 상태 유지 | 앱 재실행 시 유지 가능       |

---

# 5. 토큰 정책

| 항목              | 정책                   |
| --------------- | -------------------- |
| Access Token 만료 | Refresh Token 기반 재발급 시도 |
| Refresh 실패      | 로그인 화면 이동            |
| 로그아웃            | 토큰 및 사용자 상태 초기화      |
| 인증 실패 API       | 401 응답 처리            |

## 토큰 저장 방식 (MVP 확정)

| 토큰 | 저장 위치 |
| --- | --- |
| Access Token | 메모리(in-memory) |
| Refresh Token | `httpOnly Secure SameSite=Strict` cookie |

- Refresh Token은 서버가 cookie로 발급하며, 자동 로그인은 앱 재시작 시 cookie를 이용해 `/auth/refresh`를 호출하여 처리한다
- Access Token은 메모리에만 보관하며 localStorage, IndexedDB, sessionStorage에 저장하지 않는다
- 로그아웃 시 서버에서 Refresh Token cookie를 만료 처리하고, 프론트에서 메모리의 Access Token을 삭제한다
