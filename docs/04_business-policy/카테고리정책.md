# 카테고리정책

## 1. 문서 목적

본 문서는 카테고리 관리 및 거래 연동 기준을 정의한다.

---

# 2. 카테고리 기본 정책

| 항목 | 정책 |
| --- | --- |
| 카테고리 종류 | 수입/지출로 구분하지 않음 |
| 카테고리 구조 | 1Depth 구조 |
| 기본 카테고리 | 모든 회원에게 동일하게 제공하는 공통 데이터 |
| 사용자 카테고리 | 회원이 새로 등록한 데이터만 사용자별로 관리 |
| 사용자 카테고리 수정 | 명칭, 아이콘, 표시 여부 수정 가능 |
| 기본 카테고리 수정 | 명칭과 아이콘 수정 불가 |
| 아이콘 지정 | 가능 |
| 통계 포함 여부 | 사용자별로 관리하며 기본값은 포함 |

---

# 3. 표시 정책

| 항목 | 정책 |
| --- | --- |
| 표시 여부 | show=true만 신규 거래 선택 목록에 표시 |
| 기본 정렬 | 등록 순서 기준 |
| 표시 OFF | 신규 거래 선택 목록 제외 |
| 기존 거래 | 숨김 카테고리도 기존 거래 상세와 목록에서는 계속 표시 |

---

# 4. 거래 연동 정책

| 항목         | 정책         |
| ---------- | ---------- |
| 거래 등록      | 카테고리 선택 필수 |
| 삭제된 카테고리   | 기존 거래 유지   |
| 거래 연결 카테고리 | 물리 삭제 금지   |

---

# 5. 통계 포함 정책

| 항목 | 정책 |
| --- | --- |
| 기본값 | 모든 카테고리는 통계에 포함 |
| 제외 설정 | 사용자가 카테고리별로 `include_in_statistics=false`를 설정하면 통계 집계에서 제외 |
| 적용 범위 | 월별, 기간별, 카테고리별, 요약, Top 5, 히트맵, Sankey 등 모든 통계 집계 |
| 미적용 범위 | 계좌 잔액, 카드/계좌 거래내역, 거래 검색, 최근 거래, 거래 상세 |
| 표시 여부와 관계 | `show`와 독립적으로 동작. `show=false`여도 통계 포함 가능하고, `show=true`여도 통계 제외 가능 |
| 거래 데이터 | 통계 제외 설정은 거래 원본을 수정하지 않음 |

기본 카테고리와 사용자 카테고리 모두 사용자별 통계 포함 여부를 가질 수 있다. 사용자별 설정 행이 없으면 `include_in_statistics=true`로 간주한다.

---

# 6. 삭제 정책

| 항목 | 정책 |
| --- | --- |
| 삭제 처리 | 물리 삭제하지 않고 show=false 사용 |
| 기본 카테고리 | 숨김 가능, 명칭/아이콘 수정 불가, 물리 삭제 불가 |
| 사용자 카테고리 | 숨김 가능, 물리 삭제 불가 |
| Soft Delete | 우선 적용 |

---

# 7. 기본 카테고리 시드

기본 카테고리는 모든 회원에게 동일하게 제공하는 공통 데이터이다. 회원이 새로 등록하는 카테고리만 사용자별 데이터로 관리한다.

기본 카테고리는 수입/지출 유형을 분리하지 않는다.

| 순서 | 카테고리명 |
| ---: | --- |
| 1 | 급여 |
| 2 | 수입 |
| 3 | 지출 |
| 4 | 쇼핑 |
| 5 | 교통비 |
| 6 | 점심 |
| 7 | 식비 |
| 8 | 의료 |
| 9 | 교육 |
| 10 | 문화 |
| 11 | 공과금 |
| 12 | 구독 |
| 13 | 기타지출 |

정렬은 항상 등록 순서를 따른다.

---

# 8. 기본 카테고리 사용자별 숨김 기준

기본 카테고리 자체는 공통 데이터로 유지한다. 사용자가 기본 카테고리를 숨기는 경우 기본 카테고리 행을 수정하지 않고 사용자별 숨김 상태를 별도로 저장한다.

사용자별 숨김 상태는 신규 거래 등록의 카테고리 선택 목록에만 적용한다. 이미 해당 카테고리가 사용된 기존 거래에서는 카테고리명과 아이콘을 계속 표시한다.
