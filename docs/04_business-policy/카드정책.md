# 카드정책

## 1. 문서 목적

본 문서는 카드 등록 및 카드 거래 처리 기준을 정의한다.

---

# 2. 카드 기본 정책

| 항목       | 정책           |
| -------- | ------------ |
| 카드 등록    | 사용자 직접 등록    |
| 카드 사용 여부 | ON/OFF 지원    |
| 카드 아이콘   | 지정 가능        |
| 목록 표시    | 사용중 카드 우선 표시 |

---

# 3. 거래 연동 정책

| 항목    | 정책                     |
| ----- | ---------------------- |
| 거래 저장 | `transaction_type=EXPENSE`, `wallet_type=CARD` 기준 저장 |
| 수입 거래 | 카드는 수입이 없으므로 수입 선택 시 카드 선택 불가 |
| 통계 반영 | 카드 거래는 계좌 거래와 독립 관리하며 지출 통계에 포함 가능 |
| 카드 거래 | TRANSACTION 기준 관리      |
| 계좌 잔액 | 카드 거래 저장/수정/삭제 시 계좌 잔액 변경 없음 |

---

# 4. 삭제 정책

| 항목       | 정책          |
| -------- | ----------- |
| 거래 연결 카드 | 물리 삭제 금지    |
| 삭제 처리    | use_yn=false 처리 |
| 비활성 카드   | 선택 목록 제외    |

## 4.1 비활성 처리 기준

카드를 더 이상 사용하지 않으려는 경우 `use_yn=false`로 비활성화한다. 비활성화된 카드는 거래 등록 선택 목록에서 제외하며, 기존 카드 거래 내역과 통계 산출 기준은 유지한다.

## 4.2 아카이브(영구 삭제) 처리 기준

카드는 `DELETE /api/v1/cards/{id}` API로 아카이브 처리할 수 있다. 아카이브된 카드는 `deleted_yn=true`로 변경되며 카드 목록에서 완전히 제외한다.

아카이브 시 `delete_transactions` 옵션에 따라 연결 거래를 함께 소프트 삭제하거나 보존할 수 있다.

- `delete_transactions=false` (기본값): 기존 거래 내역은 보존하며, 해당 거래는 조회 및 삭제만 가능하고 수정은 불가
- `delete_transactions=true`: 연결된 모든 거래를 `deleted_yn=true`로 소프트 삭제

아카이브된 카드의 카드명은 동일 사용자 내에서 재등록 시 사용할 수 있다.

카드와 계좌는 MVP에서 결제 계좌로 직접 연결하지 않는다. 카드 거래는 `wallet_type=CARD`, `wallet_id=card_id` 기준으로 TRANSACTION에서 독립 관리한다.
