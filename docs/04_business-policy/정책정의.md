# 8. 정책 정의

## 8.1 회원 정책

| 항목        | 정책 내용                                    |
| --------- | ---------------------------------------- |
| 회원가입 방식   | 이메일 기반 회원가입 방식 사용                        |
| 이메일 중복    | 동일 이메일 중복 가입 불가 (탈퇴 후 즉시 재가입 가능)         |
| 비밀번호 정책   | 최소 8자 이상 입력 권장                           |
| 비밀번호 저장   | 비밀번호는 암호화 후 저장                           |
| 로그인 실패 처리 | 로그인 정보 불일치 시 오류 메시지 표시                   |
| 로그인 실패 문구 | 미가입 또는 정보 불일치 모두 "가입되어 있지 않은 계정입니다." 기준  |
| 로그인 유지    | Access Token + Refresh Token 기반 인증 유지    |
| 토큰 만료 처리  | Access Token 만료 시 Refresh Token으로 재발급 시도 |
| 재발급 실패 처리 | Refresh 실패 시 로그인 화면 이동                   |
| 비밀번호 재설정  | 가입된 이메일 기준으로 비밀번호 재설정 가능                 |
| 자동 로그인    | 앱 재실행 시 로그인 상태 유지 가능                     |
| 로그아웃      | 사용자가 직접 로그아웃 가능                          |
| 로그아웃 처리   | 로그아웃 시 토큰 및 사용자 상태 초기화                   |
| 회원탈퇴 처리   | 탈퇴 시 사용자 등록 서비스 데이터 물리 삭제                |
| 회원탈퇴 재가입  | 탈퇴 후 유예기간 없이 동일 이메일로 즉시 재가입 가능           |
| 회원탈퇴 복구   | 탈퇴 후 기존 데이터 복구 불가                        |

---

# 8.2 거래(내역) 관리 정책

| 항목       | 정책 내용                            |
| -------- | -------------------------------- |
| 거래 유형    | 수입 / 지출 구분 필수                    |
| 지갑 유형    | 계좌 / 카드 결제수단 구분 필수              |
| 카드 수입    | 수입 선택 시 카드 선택 불가                |
| 금액 입력    | 숫자만 입력 가능                        |
| 금액 조건    | 0 이하 금액 입력 불가                    |
| 날짜 입력    | 사용자가 직접 날짜 선택 가능 (`transaction_date`는 사용자 로컬 캘린더 날짜 기준) |
| 미래 날짜    | 미래 날짜 등록 불가 (사용자 timezone 기준 today 이후 날짜 차단) |
| 거래 저장 기준 | 계좌 거래는 잔액 반영, 카드 거래는 카드 내역으로 저장 |
| 거래 수정 정책 | 계좌 거래는 기존 반영 금액 복구 후 변경 금액 기준 재계산 |
| 거래 삭제 정책 | 계좌 거래는 잔액 복구, 카드 거래는 내역 삭제 처리 |
| 거래 상태 관리 | 삭제 데이터는 `deleted_yn` 기준 논리 삭제 처리 |
| 메모 입력    | 메모는 선택 입력                        |
| 상세 조회    | 모든 거래는 상세 화면 조회 가능               |
| 검색 조건    | 기간, 키워드 검색 지원                    |
| 정렬 기준    | 거래일자 최신순 우선 정렬                   |
| 오프라인 저장  | 오프라인 상태에서도 거래 임시 저장 가능           |
| 동기화 처리   | 네트워크 복구 시 서버 데이터와 동기화 수행         |

---

# 8.3 계좌 관리 정책

| 항목     | 정책 내용                 |
| ------ | --------------------- |
| 계좌 등록  | 사용자가 직접 등록            |
| 계좌명 중복 | 동일 사용자 기준 중복 불가       |
| 초기 잔액  | 계좌 등록 시 초기 잔액 입력 가능   |
| 현재 잔액  | 거래 반영 시 자동 계산         |
| 잔액 계산  | 수입 추가 / 지출 차감 기준 계산   |
| 사용 여부  | 사용여부 ON/OFF 지원        |
| 비활성 처리 | use_yn = false 기준 관리      |
| 아이콘 설정 | 계좌별 아이콘 지정 가능         |
| 목록 표시  | 사용 여부가 ON인 계좌 우선 표시   |
| 삭제 정책  | 사용중 계좌는 즉시 물리 삭제하지 않음 |

# 8.4 카드 관리 정책

| 항목       | 정책 내용                 |
| -------- | --------------------- |
| 카드 등록    | 사용자가 직접 등록            |
| 카드 사용 여부 | 사용여부 ON/OFF 지원        |
| 카드 아이콘   | 카드별 아이콘 지정 가능         |

| 수입 처리    | 카드는 수입 거래 결제수단으로 선택 불가 |
| 목록 표시    | 사용중 카드 우선 노출          |
| 삭제 정책    | 사용중 카드는 즉시 물리 삭제하지 않음 |
| 통계 반영    | 카드 거래는 계좌 거래와 독립적으로 관리하며 지출 통계에 포함 가능 |

# 8.5 카테고리 정책

| 항목       | 정책 내용                    |
| -------- | ------------------------ |
| 기본 카테고리 | 모든 회원에게 동일하게 제공하는 공통 데이터 |

| 사용자 카테고리 수정 | 명칭/아이콘/표시 여부 변경 가능 |
| 표시 여부    | 표시여부가 ON인 카테고리만 선택 목록 표시 |
| 기본 정렬    | 등록 순서 기준 정렬              |
| 거래 연동    | 거래 등록 시 카테고리 선택 필수       |
| 삭제 정책    | 실제 삭제 대신 show=false 처리, 기존 거래 표시는 유지 |

# 8.7 통계 정책

| 항목     | 정책 내용                      |
| ------ | -------------------------- |
| 통계 기준  | 저장된 거래 데이터 기준 산출           |

| 그래프 처리 | 월별 소비 그래프 및 카테고리 비율 그래프 제공 |
| 제외 데이터 | deleted_yn = true 데이터 제외      |

# 8.10 API 및 서버 정책

| 항목          | 정책 내용                     |
| ----------- | ------------------------- |
| API 형식      | REST API 기준 사용            |
| 인증 방식       | JWT 기반 인증 사용              |
| 응답 형식       | 공통 Response Wrapper 구조 사용 |
| 오류 응답       | HTTP 상태코드 + 오류 메시지 반환     |
| 유효성 검사      | 서버 기준 최종 검증 수행            |
| 시간 기준       | 서버 UTC 저장 후 사용자 로컬 변환     |
| 날짜 변환 규칙   | `transaction_date`는 사용자 로컬 캘린더 날짜(YYYY-MM-DD)로 저장 및 전달 |
| timestamp 기준 | `created_at`, `updated_at`, `synced_at` 등 모든 timestamp는 UTC ISO-8601 형식으로 저장 및 응답 |
| 미래 날짜 검증   | 사용자 timezone 기준 today 이후 날짜 차단. 서버는 요청에 포함된 timezone 또는 클라이언트 로컬 기준으로 검증 |
| API Timeout | 일정 시간 초과 시 실패 처리          |
| 재시도 정책      | 네트워크 오류 시 재시도 가능          |
