# 동기화정책

## 1. 문서 목적

본 문서는 오프라인 저장 데이터와 서버 데이터 동기화 기준을 정의한다.

---

# 2. 동기화 기본 정책

| 항목       | 정책        |
| -------- | --------- |
| 동기화 시점   | 네트워크 복구 시 |
| 자동 동기화   | 지원        |
| 수동 재시도   | 가능        |
| 상태 기반 처리 | 적용        |
| 서버 클라이언트 식별 | `SYNC_CLIENT.client_id` 기준 |
| 서버 반영 이력 | `SYNC_HISTORY` 기록 |

---

# 3. 동기화 대상

| 대상    | 처리             |
| ----- | -------------- |
| 거래 등록 | 서버 반영          |
| 거래 수정 | 최신 상태 반영       |
| 거래 삭제 | deleted_yn 동기화 |

MVP 기준 동기화 대상은 거래만 포함한다.

| 대상 | 처리 |
|---|---|
| 사용자 설정 | 온라인 API 저장, 동기화 Queue 제외 |
| 계좌/카드/카테고리/아이콘 | 온라인 API 저장, 동기화 Queue 제외 |

---

# 4. 충돌 정책

| 항목     | 정책                |
| ------ | ----------------- |
| 충돌 기준  | updated_at 기준     |
| 최신 데이터 | 우선 적용             |
| 중복 저장  | `(user_id, sync_client_id, client_temp_id)` 기준 방지 |
| 클라이언트 식별 | `(user_id, client_id)` 기준 클라이언트 등록 |
| 마지막 동기화 | `SYNC_CLIENT.last_synced_at` 갱신 |

---

# 5. 실패 정책

| 항목      | 정책          |
| ------- | ----------- |
| 동기화 실패  | 재시도 대기      |
| 서버 오류   | 오류 메시지 표시   |
| 네트워크 오류 | 오프라인 상태 유지  |
| 부분 실패   | 실패 데이터만 재시도 |

---

# 6. 데이터 무결성 정책

| 항목               | 정책 |
| ---------------- | -- |
| user_id 검증       | 필수 |
| category_id 검증   | 필수 |
| wallet_id 검증     | 필수 |
| deleted_yn 동기화   | 유지 |
| 최신 updated_at 유지 | 필수 |
| client_temp_id 유지 | 필수 |
| sync_client_id 저장 | 필수 |
| synced_at 기록 | 필수 |
| 서버 반영 이력 | 성공/실패/충돌 모두 기록 |
