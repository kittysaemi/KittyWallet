# 계좌정책

## 1. 문서 목적

본 문서는 계좌 등록 및 잔액 관리 기준을 정의한다.

# 2. 계좌 기본 정책

| 항목     | 정책              |
| ------ | --------------- |
| 계좌 등록  | 사용자 직접 등록       |
| 계좌명 중복 | 동일 사용자 기준 중복 불가. 단, 아카이브된 계좌명(`deleted_yn=true`)은 재사용 가능 |

| 마이너스 한도 | 0 이상 숫자로 계좌별 설정. 감소 시 기존 거래 잔액 흐름 검증 |
| 사용 여부  | ON/OFF 지원       |

# 3. 잔액 정책

| 항목       | 정책                                         |
| -------- | ------------------------------------------ |
| 수입 거래    | 잔액 증가                                     |
| 지출 거래    | 잔액 감소                                     |
| 거래 수정    | 차액 기준 재계산                                 |
| 지출 거래 삭제 | 잔액 복구 처리. 날짜별 잔액 검증 생략 (잔액 증가 방향) |
| 수입 거래 삭제 | 잔액 복구 처리. 삭제 후 날짜별 잔액 흐름 검증. 허용 범위 아래로 내려가면 삭제 차단 |
| 잔액 음수 허용 | 기본값은 허용하지 않음. `allow_negative_balance=true` 계좌만 허용 |
| 마이너스 한도 | `negative_balance_limit` 기준 최소 허용 잔액은 `-negative_balance_limit` |
| 잔액 부족 처리 | 거래 등록/수정, 수입 삭제, 마이너스 설정 변경 반영 후 날짜별 잔액이 허용 범위 아래로 내려가는 경우 사용자에게 알리고 저장 차단 |

## 3.1 마이너스 설정 변경 정책

`allow_negative_balance` 또는 `negative_balance_limit`를 변경할 때는 변경 전 기존 거래 잔액 흐름을 새 설정 기준으로 검증한다.

| 변경 유형 | 검증 여부 |
|---|---|
| `allow_negative_balance` OFF → ON | 검증 (마이너스 한도 새로 적용) |
| `allow_negative_balance` ON → OFF | 검증 (기존 마이너스 잔액 불가) |
| `negative_balance_limit` 감소 | 검증 (기존 마이너스 잔액이 새 한도 초과 가능) |
| `negative_balance_limit` 증가 | 검증 (한도 완화이므로 실질적으로 차단 안 됨) |

기존 날짜별 잔액 흐름이 새 설정의 최소 허용 잔액을 위반하면 설정 변경을 차단한다.

## 3.2 날짜별 잔액 검증

계좌 잔액 검증은 단순 최종 `current_balance`가 아니라 `initial_balance`에서 시작해 삭제되지 않은 계좌 거래를 `transaction_date` 오름차순으로 일자별 합산한 잔액 흐름을 기준으로 한다.

같은 날짜의 거래는 별도 시간이 없으므로 수입/지출을 일자별로 합산한 뒤 일별 잔액으로 검증한다.

| 계좌 설정 | 허용 기준 |
|---|---|
| `allow_negative_balance=false` | 어느 날짜의 계산 잔액도 0 미만 불가 |
| `allow_negative_balance=true` | 어느 날짜의 계산 잔액도 `-negative_balance_limit`보다 작아질 수 없음 |

거래 수정 시 기존 거래를 제거하고 변경 거래를 반영한 가상 상태로 검증한다. 거래 계좌가 변경되면 기존 계좌와 새 계좌의 날짜별 잔액 흐름을 모두 검증한다.

오프라인 저장 거래는 클라이언트 Queue 기준으로 1차 검증하고, 서버 동기화 시 서버가 동일 정책으로 최종 검증한다.

# 5. 비활성 정책

| 항목       | 정책          |
| -------- | ----------- |
| use_yn=false | 선택 목록 제외 가능 |
| 거래 연결 계좌 | 물리 삭제 금지    |
| 삭제 대신    | 비활성 처리 사용   |

## 5.1 비활성 처리 기준

계좌를 더 이상 사용하지 않으려는 경우 `use_yn=false`로 비활성화한다. 비활성화된 계좌는 거래 등록 선택 목록에서 제외하며, 기존 거래 내역과 잔액 계산 기준은 유지한다.

## 5.2 아카이브(영구 삭제) 처리 기준

계좌는 `DELETE /api/v1/accounts/{id}` API로 아카이브 처리할 수 있다. 아카이브된 계좌는 `deleted_yn=true`로 변경되며 계좌 목록에서 완전히 제외한다.

아카이브 시 `delete_transactions` 옵션에 따라 연결 거래를 함께 소프트 삭제하거나 보존할 수 있다.

- `delete_transactions=false` (기본값): 기존 거래 내역은 보존하며, 해당 거래는 조회 및 삭제만 가능하고 수정은 불가
- `delete_transactions=true`: 연결된 모든 거래를 `deleted_yn=true`로 소프트 삭제

아카이브된 계좌의 계좌명은 동일 사용자 내에서 재등록 시 사용할 수 있다.
