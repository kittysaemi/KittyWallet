# 시간대 정책

## 1. 문서 목적

본 문서는 미냥이 지갑 서비스에서 날짜 계산의 기준이 되는 시간대(timezone)를 사용자가 직접 설정하는 기능의 정책을 정의한다.

---

## 2. 배경 및 원칙

서버는 모든 timestamp(`created_at`, `updated_at`, `synced_at`)를 UTC로 저장한다.

사용자가 가계부에 입력하는 **거래 날짜(`transaction_date`)** 및 **"오늘"을 기준으로 한 날짜 계산**(미래 날짜 검증, 통계 기본 월, 대시보드 기준일 등)은 사용자의 시간대(timezone) 기준으로 처리해야 한다.

시간대가 잘못 설정된 경우, 자정 이후 날짜가 갱신되지 않거나 오늘 날짜를 "미래 날짜"로 오인하는 문제가 발생한다.

---

## 3. 시간대 설정

### 3.1 기본값

| 항목 | 값 |
|---|---|
| 기본 시간대 | `Asia/Seoul` (KST, UTC+9) |
| 설정 저장 위치 | `USER_SETTING` 테이블, `setting_key = "timezone"` |

### 3.2 지원 시간대 목록

| 표시명 | IANA 식별자 | UTC 오프셋 |
|---|---|---|
| 한국 (KST) | `Asia/Seoul` | UTC+9 |

> 추후 서비스 확장 시 지원 시간대를 추가한다.

### 3.3 설정 변경 규칙

| 항목 | 정책 |
|---|---|
| 변경 적용 시점 | 앱 설정 저장 즉시 적용 |
| 기존 거래 데이터 영향 | 없음 (`transaction_date`는 사용자가 직접 입력한 날짜이므로 변경 불필요) |
| 기존 timestamp 영향 | 없음 (UTC로 저장되며 timezone 설정과 무관) |
| 미설정 시 기본값 | `Asia/Seoul` |

---

## 4. 날짜 계산 기준

### 4.1 timezone 적용 대상

timezone 설정은 "오늘"을 계산하는 모든 곳에 적용된다.

| 위치 | 적용 내용 |
|---|---|
| 거래 등록 폼 날짜 기본값 | 설정된 timezone 기준 오늘 날짜 |
| 거래 등록/수정 미래날짜 검증 | 설정된 timezone 기준 today 이후 차단 |
| 거래 검색 기본 날짜 범위 | 설정된 timezone 기준 오늘/3개월 전 |
| 거래 내역 월 탐색 초기값 | 설정된 timezone 기준 현재 년/월 |
| 통계 기본 월 | 설정된 timezone 기준 현재 월 (`YYYY-MM`) |
| 대시보드 기준일 | 설정된 timezone 기준 오늘 날짜 |

### 4.2 timezone 비적용 대상 (UTC 유지)

| 데이터 | 이유 |
|---|---|
| `created_at`, `updated_at`, `synced_at` | 서버-클라이언트 간 timestamp 기준. 시간대와 무관하게 UTC 저장 |
| 토큰 만료 시각 | 서버 기준 절대 시간. timezone 설정과 무관 |
| 동기화 시각 | 서버 기준 절대 시간 |

---

## 5. 프론트엔드 날짜 유틸리티 기준

오늘 날짜를 계산할 때는 반드시 사용자 설정 timezone을 사용한다.

```ts
// 올바른 방법 (설정된 timezone 사용)
const today = new Date().toLocaleDateString("sv-SE", { timeZone: userTimezone });
// → "2026-06-14" 형식 반환 (YYYY-MM-DD)

// 잘못된 방법 (UTC 기준 - 사용 금지)
const today = new Date().toISOString().split("T")[0];
```

`sv-SE` 로케일은 `YYYY-MM-DD` 형식을 반환하므로 별도 파싱 없이 API 요청에 직접 사용 가능하다.

---

## 6. 백엔드 미래날짜 검증 기준

거래 등록/수정 API는 요청에 포함된 `timezone` 값을 기준으로 미래 날짜를 검증한다.

```ts
// 올바른 방법 (요청의 timezone 사용)
const todayStr = new Date().toLocaleDateString("sv-SE", { timeZone: requestTimezone });

// 잘못된 방법 (UTC 기준 - 사용 금지)
const todayStr = new Date().toISOString().split("T")[0];
```

`timezone` 미전달 시 기본값 `"Asia/Seoul"` 을 사용한다.
