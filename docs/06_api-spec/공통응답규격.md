# 공통 응답 규격

## 성공 응답

```json
{
  "success": true,
  "data": {},
  "error": null
}
```

## 실패 응답

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "ERROR_CODE",
    "message": "오류 메시지",
    "details": {}
  }
}
```

---

# HTTP 상태 코드

| 상태코드 | 설명 |
|---|---|
| 200 | 정상 처리 |
| 201 | 생성 완료 |
| 400 | 요청 오류 |
| 401 | 인증 실패 |
| 403 | 권한 없음 |
| 404 | 데이터 없음 |
| 409 | 중복 데이터 |
| 500 | 서버 오류 |

---

# Error Code → HTTP Status 매핑

> 개별 오류 코드의 목록과 설명은 `오류코드정의.md`(코드 카탈로그)를 참조한다.
> 본 표는 각 코드가 어떤 HTTP Status로 응답되는지, 프론트엔드 공통 API Client에서 어떻게 처리하는지를 정의한다.
