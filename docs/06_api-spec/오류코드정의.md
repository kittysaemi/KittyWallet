## 화면 오류 표시 정책

서버 API 오류는 응답의 `error.code`를 유지한다. 등록 화면을 포함한 사용자 화면은 서버의 상세 `error.message`를 직접 표시하지 않고 다음 공통 형식을 사용한다.

```text
오류입니다 #{error.code} 관리자에 문의하세요.
```

HTTP 응답을 받지 못한 클라이언트 오류는 `NETWORK_001`, 오프라인 등록 처리 실패는 `OFFLINE_001`, 식별할 수 없는 오류는 `INTERNAL_001`로 표시한다. 사용자가 즉시 수정할 수 있는 입력 필드 검증 메시지는 이 정책의 대상이 아니다.
