# 서비스기획

## 1. 서비스 개요

미냥이 지갑은 사용자가 직접 수입, 지출, 카드 사용 내역을 기록하고 관리할 수 있는 PWA 기반 모바일 가계부 서비스이다.

서비스는 모바일 환경에서 빠르게 거래를 기록하고 소비 흐름을 확인할 수 있도록 설계한다.

주요 목표:

* 빠른 거래 입력
* 직관적인 모바일 UI
* 계좌 및 카드 통합 관리
* 소비 흐름 분석
* 오프라인 대응 가능한 구조
* 유지보수 가능한 구조

---

# 2. 서비스 목적

## 2.1 빠른 거래 기록 환경 제공

사용자가 모바일 환경에서 최소 입력만으로 빠르게 거래를 기록할 수 있도록 한다.

지원 목표:

* 빠른 금액 입력
* 직관적인 카테고리 선택
* 최소 단계 입력 구조
* 모바일 터치 중심 UI

---

## 2.2 소비 흐름 관리 지원

사용자가 월별 소비 흐름과 카테고리별 지출 현황을 쉽게 확인할 수 있도록 한다.

지원 목표:

* 월별 수입/지출 통계
* 카테고리별 소비 분석
* 최근 거래 조회
* 기간별 거래 검색

---

## 2.3 모바일 앱 수준 사용자 경험 제공

웹 기반 서비스이지만 모바일 앱 수준 사용자 경험 제공을 목표로 한다.

지원 목표:

* 홈 화면 설치
* 전체화면 사용
* Safe Area 대응
* 빠른 화면 전환
* 오프라인 대응

---

# 3. 핵심 서비스 구성

| 기능      | 설명                  |
| ------- | ------------------- |
| 거래 관리   | 수입/지출 거래 등록 및 계좌/카드 결제수단 관리 |
| 계좌 관리   | 계좌 등록 및 잔액 관리       |
| 카드 관리   | 카드 등록 및 카드 거래 관리    |
| 카테고리 관리 | 거래 분류 기준 관리         |
| 아이콘 관리  | 계좌/카드/카테고리 아이콘 관리   |
| 통계 기능   | 소비 흐름 및 월별 통계 제공    |

---

# 4. 서비스 구조 방향

## 4.1 거래 중심 구조

서비스는 거래 데이터를 중심으로 구성한다.

구분 기준:

* transaction_type
* wallet_type

---

## 4.2 사용자 기준 데이터 분리

모든 데이터는 user_id 기준으로 사용자별 분리 저장한다.

---

## 4.3 정책 기반 개발 구조

모든 기능은 정책 정의를 기준으로 개발한다.

포함 기준:

* 상태 처리
* 예외 처리
* 오류 처리
* 데이터 무결성
* Soft Delete 정책

---

# 5. PWA 및 하이브리드앱 방향

## 5.1 PWA 기반 서비스 구성

서비스는 PWA 기반 모바일웹 구조를 기본으로 사용한다.

지원 기능:

* 홈 화면 설치
* 캐시 기반 접근
* 오프라인 지원
* Service Worker 사용

---

## 5.2 하이브리드앱 확장 가능 구조 유지

향후 Android 및 iOS 앱 확장을 고려하여 하이브리드앱 전환이 가능한 구조를 유지한다.

적용 방향:

* 모바일 우선 UI 유지
* WebView 대응 구조 유지
* Native 기능 확장 가능 구조 유지

---

# 6. 최종 서비스 목표

미냥이 지갑은 다음 목표를 만족하는 서비스를 지향한다.

* 모바일 앱 수준 사용자 경험
* 빠른 거래 입력 환경
* 유지보수 가능한 구조
* 확장 가능한 구조
* 오프라인 대응 가능한 PWA
* 정책 기반 개발 구조
* AI 협업 가능한 프로젝트 구조
