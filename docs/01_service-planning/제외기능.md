# 제외기능

## 1. 문서 목적

본 문서는 미냥이 지갑 프로젝트의 초기 개발 범위에서 제외되는 기능을 정의한다.

제외 기능은 향후 확장 가능성을 고려하되, 초기 MVP 범위에서는 개발하지 않는다.

---

# 2. 금융기관 연동 기능 제외

| 기능        | 제외 사유              |
| --------- | ------------------ |
| 은행 API 연동 | 초기 개발 복잡도 증가       |
| 자동 거래 수집  | 외부 연동 비용 및 유지보수 부담 |
| 실시간 계좌 조회 | 금융 연동 구조 미포함       |
| 카드사 연동    | 인증 및 보안 부담 증가      |

---

# 3. 고급 가계부 기능 제외

| 기능       | 제외 사유       |
| -------- | ----------- |
| 예산 관리    | MVP 우선순위 제외 |
| 반복 거래    | 초기 구조 단순화   |
| 목표 저축 기능 | 핵심 기능 우선 개발 |
| 자산 분석    | MVP 범위 외    |
| 투자 관리    | 서비스 방향 범위 외 |

---

# 4. 협업 기능 제외

| 기능        | 제외 사유         |
| --------- | ------------- |
| 공유 가계부    | 사용자 구조 복잡도 증가 |
| 가족 계정     | 권한 구조 추가 필요   |
| 다중 사용자 협업 | MVP 범위 외      |

---

# 5. 고급 알림 기능 제외

| 기능                | 제외 사유           |
| ----------------- | --------------- |
| Push Notification | 초기 Native 기능 제외 |
| 소비 알림             | 우선순위 낮음         |
| 카드 결제일 알림         | MVP 범위 외        |

---

# 6. Native 기능 제외

| -------- | --------------- |

# 7. 고급 통계 기능 제외

| 기능       | 제외 사유     |
| -------- | --------- |

# 8. 운영 및 관리자 기능 제외

| 기능        | 제외 사유         |
| --------- | ------------- |
| 관리자 페이지   | 단일 사용자 서비스 우선 |
| 운영 통계 페이지 | MVP 범위 외      |
| 공지 관리     | 초기 운영 규모 미적용  |

---

# 9. 향후 확장 가능 기능

다음 기능은 향후 확장 가능성을 고려한다.

* 금융기관 연동
* Push Notification
* 반복 거래
* 예산 관리
* 공유 가계부
* Native 기능
* 앱스토어 배포
* AI 소비 분석

초기 MVP에서는 제외하지만 구조적으로 확장 가능하도록 설계한다.
