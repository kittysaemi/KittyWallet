# MVP범위

## 1. 문서 목적

본 문서는 미냥이 지갑 프로젝트의 초기 MVP(Minimum Viable Product) 범위를 정의한다.

MVP는 실제 사용 가능한 최소 기능 제공을 목표로 하며, 핵심 기능 중심으로 우선 개발한다.

---

# 2. MVP 목표

초기 MVP는 다음 목표를 중심으로 구성한다.

* 거래 기록 기능 제공
* 소비 흐름 확인 기능 제공
* 모바일 중심 사용자 경험 제공
* PWA 기반 설치 지원
* 오프라인 기본 대응 지원

---

# 3. MVP 포함 기능

## 3.1 회원 기능

| 기능     | 설명          |
| ------ | ----------- |
| 회원가입   | 이메일 기반 회원가입 |

| 기능    | 설명                |
| ----- | ----------------- |

# 4. MVP 기술 범위

| 영역       | 적용 범위                 |
| -------- | --------------------- |
| Frontend | React + TypeScript    |
| Backend  | NestJS                |
| Database | PostgreSQL            |
| ORM      | Prisma                |
| 상태 관리    | Zustand + React Query |
| 인증       | JWT                   |
| PWA      | Vite PWA Plugin       |

---

# 5. MVP 운영 기준

| 항목     | 기준                                |
| ------ | --------------------------------- |
| 플랫폼    | 모바일 우선                            |
| 데이터 구조 | 거래 중심 구조                          |
| 상태 처리  | loading / empty / error / offline |
| 삭제 처리  | Soft Delete 우선                    |
| API 구조 | REST API                          |
| 인증 구조  | JWT 기반 인증                         |

---

# 6. MVP 완료 기준

다음 조건 충족 시 MVP 완료로 판단한다.

* 회원 기능 정상 동작
* 거래 저장 및 조회 가능
* 계좌/카드/카테고리 관리 가능
* 통계 기능 정상 동작
* 모바일 환경 정상 사용 가능
* PWA 설치 가능
* 오프라인 기본 대응 가능
