# 화면 정의

## 1. 화면 정의 개요

본 문서는 미냥이 지갑의 화면 구성, 상태 처리, 정책, 예외 처리 및 API 연결 기준을 정의한다.

미냥이 지갑은 브라우저 기반 PWA 모바일 웹앱 서비스이며, 모바일 브라우저 환경에서 실행 가능하도록 설계한다.

또한 홈 화면 설치를 통해 앱 형태로 사용할 수 있으며, 향후 Capacitor 기반 하이브리드 앱 구조로 확장하는 것을 목표로 한다.

화면 정의는 단순 UI 설명이 아니라 실제 개발 가능한 기준 문서를 목표로 한다.

모든 화면은 다음 기준을 포함해야 한다.

* 화면 설명
* 구성 요소
* 주요 기능
* 상태 정의
* 정책 정의
* 예외 처리
* API 연결
* 모바일 UX 고려사항
* PWA 환경 고려사항

---

# 2. 공통 화면 규칙

## 2.1 공통 상태 정의

화면 상태의 canonical 기준은 `docs/03_screen-spec/상태정의.md`를 따른다.

모든 화면은 필요한 경우 다음 UI State를 사용한다.

| 상태      | 설명         |
| ------- | ---------- |
| loading | API 호출 중   |
| empty   | 데이터 없음     |
| error   | 오류 발생      |
| saving  | 저장 요청 중    |

네트워크, 인증, PWA, 캐시, 동기화 상태는 화면이 직접 소유하지 않고 `상태정의.md`의 상태 소유자 기준에 따라 참조한다.

| 상태          | 설명           |
| ----------- | ------------ |
| refreshing  | 새로고침 중       |
| offline     | 네트워크 연결 없음   |
| pending_sync | 동기화 대기       |
| syncing     | 동기화 진행 중     |
| sync_failed | 동기화 실패       |
| expired     | 인증 만료 상태     |
| installable | PWA 설치 가능 상태 |

# 6. 비밀번호 재설정 화면

## 화면 설명

비밀번호 재설정은 다음 2단계 플로우로 처리한다.

1. **재설정 요청 화면**: 이메일 입력 후 재설정 링크 발송 요청
2. **비밀번호 변경 화면**: 이메일의 재설정 링크 클릭 → URL에 포함된 `reset_token`을 자동 인식 후 새 비밀번호 입력

---

## 6-1. 재설정 요청 화면

### 구성 요소

| 구성 요소     | 설명         |
| --------- | ---------- |
| 이메일 입력    | 사용자 이메일 입력 |
| 요청 버튼     | 재설정 링크 발송 요청 |
| 로그인 이동 버튼 | 로그인 화면 이동  |

### 주요 기능

* 이메일 입력 후 `POST /api/v1/auth/request-reset-password` 호출
* 성공 시 "이메일을 확인해주세요" 안내 메시지 표시
* 계정 존재 여부와 관계없이 동일한 성공 메시지 표시 (계정 노출 방지)

### 예외 처리

* 이메일 형식 오류 (VALIDATION_001)
* 재설정 요청 횟수 초과 (AUTH_010)
* 네트워크 오류

---

## 6-2. 비밀번호 변경 화면

### 화면 진입

재설정 이메일의 링크 클릭 시 URL에 `reset_token`이 파라미터로 포함된다.
예: `/reset-password?token={reset_token}`

프론트는 URL에서 `reset_token`을 추출하여 API 요청에 포함한다.

### 구성 요소

| 구성 요소         | 설명              |
| ------------- | --------------- |
| 새 비밀번호 입력    | 새 비밀번호 입력       |
| 새 비밀번호 확인 입력 | 비밀번호 확인         |
| 저장 버튼         | 비밀번호 변경 요청      |
| 로그인 이동 버튼     | 로그인 화면 이동       |

### 주요 기능

* URL에서 `reset_token` 추출
* `reset_token` 유효성 확인 (만료 또는 없으면 재설정 요청 화면으로 이동)
* 새 비밀번호 입력 후 `POST /api/v1/auth/reset-password` 호출
* 성공 시 로그인 화면으로 이동

### 상태 정의

| 상태      | 설명       |
| ------- | -------- |
| loading | 재설정 요청 중 |
| error   | 재설정 실패   |
| offline | 네트워크 미연결 |

### 예외 처리

* `reset_token` 없음 또는 URL 파라미터 누락 → 재설정 요청 화면으로 이동
* 만료된 `reset_token` (AUTH_007) → 재설정 재요청 안내
* validation 실패 (VALIDATION_001, AUTH_005)
* 네트워크 오류

---

## API 연결

| 기능              | API                                         |
| --------------- | ------------------------------------------- |
| 재설정 링크 요청       | POST /api/v1/auth/request-reset-password    |
| 비밀번호 재설정        | POST /api/v1/auth/reset-password            |

# 8. 거래 등록 화면

## 화면 설명

수입 및 지출 거래를 한 화면에서 등록 및 수정하는 화면이다.

거래 유형은 수입/지출만 제공하고, 계좌/카드는 지갑으로 선택한다.

---

## 구성 요소

| 구성 요소    | 설명       |
| -------- | -------- |
| 거래 유형 선택 | 수입 / 지출  |
| 금액 입력    | 거래 금액 입력 |
| 날짜 입력    | 거래 날짜 선택 |
| 지갑 선택 | 계좌 / 카드 지갑 선택 |
| 계좌 선택    | 지갑 유형이 계좌일 때 계좌 선택 |
| 카드 선택    | 지갑 유형이 카드일 때 카드 선택 |
| 카테고리 선택  | 카테고리 선택  |
| 메모 입력    | 메모 입력    |
| 저장 버튼    | 거래 저장 요청 |

---

## 주요 기능

* 거래 등록
* 거래 수정
* 거래 삭제
* 입력값 검증
* 날짜 검증
* 오프라인 임시 저장

---

## 상태 정의

| 상태      | 설명         |
| ------- | ---------- |
| loading | 저장 요청 중    |
| error   | 저장 실패      |
| offline | 오프라인 저장 상태 |
| syncing | 서버 동기화 중   |

---

## 정책 정의

* 미래 날짜 등록 불가
* 금액 필수 입력
* 카테고리 필수 선택
* 거래 유형은 수입/지출만 선택 가능
* 계좌/카드는 지갑이며 `wallet_type`으로 저장
* 수입 선택 시 카드 선택은 비활성화하고 계좌만 선택 가능
* 카드 거래는 항상 지출 거래로 저장
* 계좌 거래와 카드 거래 내역은 `wallet_type` 기준으로 분리 조회 가능
* 삭제 시 soft delete 처리
* 아카이브된 지갑(`wallet_deleted=true`)의 거래는 상세 조회는 가능하나 수정 화면 진입 시 상세 화면으로 리다이렉트
* 아카이브된 지갑의 거래 상세 화면에서는 수정 불가 안내 표시 및 삭제 버튼 제공

---

## 예외 처리

* validation 실패
* 저장 실패
* 네트워크 오류
* 중복 저장 요청
* 수입 선택 상태에서 카드 선택 시도
* 계좌/카드가 하나도 없거나 모두 비활성 상태인 경우 저장 불가 안내 및 계좌/카드 등록 화면으로 유도

---

## PWA 환경 고려사항

* 오프라인 상태에서도 거래 등록 가능 (IndexedDB 저장 후 Sync Queue 처리)
* 오프라인 시 계좌/카드/카테고리 목록은 Cache Storage에 캐시된 마지막 성공 응답 데이터 사용
* **첫 실행 오프라인 제한**: 계좌/카드/카테고리 캐시가 없는 상태에서 오프라인이면 거래 등록 불가. "네트워크 연결 후 기준 데이터를 불러와야 거래를 등록할 수 있습니다" 안내 메시지 표시

---

## 카드할부 설정 (거래 등록 화면 — 저장 전)

| 항목 | 기준 |
|---|---|
| 표시 조건 | wallet_type=CARD인 지출 거래에서만 표시 |
| 미표시 조건 | wallet_type=ACCOUNT이거나 수입 거래인 경우 표시 안 함 |
| 지갑 변경 | 카드 → 계좌로 변경 시 입력된 할부 설정 초기화. DB 영향 없음 |
| 입력 항목 | 할부 개월 수 (2개월 이상) |
| 금액 분배 | 저장 시 자동 분배. 나머지는 1회차에 합산 |

## 할부 거래 수정 제약 (저장 후)

| 항목 | 기준 |

# 10. 통계 화면

## 화면 설명

다양한 통계 유형(소비통계, 소비흐름, 월간 요약, Top 5 카테고리, 달력 히트맵, 카테고리 통계)을 선택하여 확인하는 화면이다.

대시보드에서 통계 버튼 클릭 시 진입하며, 상단 탭으로 통계 유형을 전환한다.

---

## 통계 유형 탭

탭은 2줄 구조로 고정 렌더링한다.

**첫 번째 줄:**

| 유형 ID | 유형명 | 설명 |
| ------- | ------ | ---- |
| spending | 소비통계 | 기간별(월/주) 소비 흐름 라인 차트 + 카테고리별 소비 |
| flow | 소비흐름 | 지출/수입 Sankey 다이어그램. 내부 세그먼트로 지출/수입 전환 |
| summary | 월간요약 | 수입/지출/순수입/거래 수/최고 지출 카테고리 요약 카드 |
| top5 | Top5 | 월간 지출 상위 5개 카테고리 + 수입 상위 5개 카테고리 카드 (금액, 비율, 컬러 바) |

**두 번째 줄:**

| 유형 ID | 유형명 | 설명 |
| ------- | ------ | ---- |
| heatmap | 달력 히트맵 | 날짜별 지출 규모를 색상 강도로 표시하고, 날짜 선택 시 해당일 지출 거래 목록을 표시하는 월 달력 |
| category-expense | 카테고리 통계 | 카테고리별 지출합계. #274/#278에서 구현. 자체 기간 컨트롤 사용 |

**타입 정의:**

```ts
type StatTab = "spending" | "flow" | "summary" | "top5" | "heatmap" | "category-expense";
```

**기존 URL 호환:**

- `tab=sankey`로 접근하면 `tab=flow&flow=expense`와 동일하게 처리한다.
- `tab=income-sankey`로 접근하면 `tab=flow&flow=income`과 동일하게 처리한다.

---

## 구성 요소

| 구성 요소 | 설명 |
| ------- | ---- |
| 통계 유형 탭 (2줄) | 1줄: 소비통계/소비흐름/월간요약/Top5. 2줄: 달력 히트맵/카테고리 통계. flex-wrap 대신 줄 단위로 명시적 렌더링 |
| 공통 날짜 네비게이터 | 이전/다음 이동. 탭별 사용 여부는 아래 날짜 네비게이터 정책을 따름 |
| 기간 선택 탭 (월/주) | spending, top5 유형에서만 표시 |
| 소비흐름 내부 세그먼트 | flow 탭 내부의 지출/수입 전환 컨트롤 |
| 콘텐츠 영역 | 선택된 통계 유형의 UI 표시 |
| 새로고침 버튼 | 현재 유형의 데이터 재조회 |

## 날짜 네비게이터 정책

탭별 날짜 네비게이터 사용 기준을 명확히 분리한다.

| 탭 | 네비게이터 유형 | 설명 |
| --- | --- | --- |
| spending | 월/주 공통 네비게이터 | viewMode에 따라 월 또는 주 단위 |
| top5 | 월/주 공통 네비게이터 | viewMode에 따라 월 또는 주 단위 |
| flow | 월 단위 공통 네비게이터 | 항상 월 단위 |
| summary | 월 단위 공통 네비게이터 | 항상 월 단위 |
| heatmap | 월 단위 공통 네비게이터 | 항상 월 단위 |
| category-expense | 자체 기간 컨트롤 | #274/#278에서 구현. 공통 네비게이터 숨김 |

- `category-expense` 탭에서는 공통 날짜 네비게이터를 렌더링하지 않는다.
- `category-expense`의 기간 상태(`전체/년별/월별`)는 기존 `baseDate`, `viewMode`와 분리하여 관리한다.
- `전체` 기간 선택 시 공통 월/주 네비게이터가 노출되거나 적용되어서는 안 된다.

## 소비흐름 탭 내부 구성

`flow` 탭은 지출 Sankey와 수입 Sankey를 하나의 탭 안에서 세그먼트로 전환한다.

**내부 상태:**

```ts
type FlowMode = "expense" | "income";
```

**URL 파라미터:**

```txt
/statistics?tab=flow&flow=expense   (기본값)
/statistics?tab=flow&flow=income
```

**동작:**

- `flow=expense`: 기존 지출 Sankey 화면 표시
- `flow=income`: 기존 수입 Sankey 화면 표시
- 기본값은 `expense`
- 내부 전환은 세그먼트 컨트롤(`지출 / 수입`)로 제공한다.
- 기존 `SankeyContent`, `IncomeSankeyContent`는 최대한 재사용한다.
- 기존 백엔드 API는 변경하지 않는다.

| 구성 요소 | 설명 |
| ------- | ---- |
| 지출/수입 세그먼트 컨트롤 | 소비흐름 탭 내부 전환. 기본값 지출 |
| 지출 Sankey | 총 지출 → 계좌/카드 → 전체 카테고리 흐름. 핑크 계열 |
| 수입 Sankey | 총 수입 → 계좌 → 전체 카테고리 흐름. 파란색 계열 |

## 통계 유형별 구성

### spending — 소비통계

기존 소비 통계 화면 UX를 유지한다.

| 카테고리별 소비 | 상위 5개 카테고리 목록 + 컬러 바 |

### flow — 소비흐름

| 구성 요소 | 설명 |
| ------- | ---- |

| 순수입 카드 | 수입 - 지출 |
| 거래 수 카드 | 삭제되지 않은 거래 건수 |
| 최고 지출 카테고리 카드 | 카테고리명, 아이콘, 금액 |

| 구성 요소 | 설명 |
| ------- | ---- |
| 지출 Top 5 섹션 | 지출 상위 5개 카테고리. 카테고리명, 아이콘, 금액, 비율, 컬러 바 표시. 총 지출 합계 표시 |
| 수입 Top 5 섹션 | 수입 상위 5개 카테고리. 카테고리명, 아이콘, 금액, 비율, 하늘색(`#38BDF8`) 컬러 바 표시. 총 수입 합계 표시 |

히트맵은 별도 외부 라이브러리 없이 CSS Grid 구현을 우선 사용한다.

## 정책 정의

* 삭제 거래는 모든 통계에서 제외한다.
* `include_in_statistics=false` 카테고리 거래는 모든 통계 유형의 집계에서 제외한다.
* 카테고리 통계 제외 설정은 통계 화면에만 반영하며 거래내역, 최근 거래, 계좌 잔액에는 반영하지 않는다.
* Top 5 통계에서 Top 5를 초과하는 카테고리는 `기타`로 합산한다.
* Sankey는 카테고리를 `기타`로 합산하지 않고 전체 카테고리로 세분화한다.
* 지출 흐름 Sankey와 지출 Top 5는 지출 거래 중심으로 산출한다.
* 수입 흐름 Sankey와 수입 Top 5는 수입 거래 중심으로 산출한다.
* 히트맵은 지출 금액 기준으로 색상 강도를 표시한다.
* 데이터가 없으면 각 유형별 empty UI를 표시한다.
* 모바일 360px 폭에서 겹침/잘림 없이 표시해야 한다.
* Sankey 렌더링 실패가 발생해도 통계 화면 전체가 깨지지 않아야 한다.
* 탭 구조는 `flex-wrap` 자동 줄바꿈 방식이 아니라, 1줄/2줄 구조를 명시적으로 유지하여 렌더링한다.
* `category-expense` 탭은 미구현 상태에서 탭을 노출하는 경우 사용자 혼란을 주지 않도록 준비 중 또는 비활성 처리를 명확히 한다.

## #274/#278과의 관계

이 통계 UI 정리(#297)는 `카테고리 통계` 탭 구조를 사전 준비하는 작업이다.

- **#278 (카테고리별 지출합계 UI 추가)은 #297 이후에 진행해야 한다.** #278은 이 탭 구조와 네비게이터 정책에 의존하기 때문이다.
- #274의 백엔드/API 작업(#275, #276, #277)은 이 작업과 직접 충돌이 적으므로 병렬로 진행할 수 있다.
- `category-expense` 탭 컨텐츠는 #278에서 구현한다. #297 작업 범위에서는 탭 슬롯과 네비게이터 예외 구조를 준비한다.

## 예외 처리

* 차트 또는 Sankey 렌더링 실패 → 해당 영역에만 오류 UI 표시, 다른 유형은 정상 동작
* API 실패 → 유형별 error 상태 표시 + 재시도 버튼
* 데이터 없음 → 유형별 empty UI 표시
* 오프라인 → 캐시된 마지막 데이터 표시 또는 오프라인 안내 메시지
* 기존 `tab=sankey`, `tab=income-sankey` URL 접근 → 호환 처리로 소비흐름 탭 지출/수입 모드로 리다이렉트

---

## 정책 정의

* 카테고리는 수입/지출로 구분하지 않음
* 기본 카테고리는 모든 회원에게 동일하게 제공
* 회원이 새로 등록한 카테고리만 사용자별 데이터로 관리
* 기본 카테고리는 아이콘과 이름만 작은 카드로 표시하며 명칭, 아이콘, 표시 여부 수정 불가
* 사용자 카테고리는 명칭, 아이콘, 표시 여부 수정 가능
* 기본 카테고리와 사용자 카테고리 모두 통계 포함 여부 수정 가능
* 카테고리는 삭제하지 않고 Show/Hide로 처리
* 표시 여부가 ON인 카테고리만 신규 거래 선택 화면에 표시
* 숨김 카테고리도 기존 거래에서는 계속 표시
* 통계 제외 카테고리는 모든 통계 화면의 집계에서만 제외하며 계좌 잔액, 거래내역, 최근 거래에는 영향 없음
* 표시 여부와 통계 포함 여부는 독립 설정으로 처리
* 정렬은 등록순
* icon_id 필수
* 카테고리명은 한글 기준 15자 이하
* 카테고리 등록은 별도 페이지가 아니라 목록 화면의 인라인 등록 폼에서 처리
* 카테고리 아이콘 선택은 별도 페이지 이동 없이 팝업/바텀시트에서 처리
* 사용자 카테고리 표시 여부는 별도 버튼 없이 목록 행의 텍스트/아이콘 외 영역 클릭으로 토글
* 통계 제외 관리는 카테고리 관리 화면의 `통계 제외 관리` 버튼으로 바텀시트를 열어 처리
* 통계 제외 바텀시트에서는 기본/사용자 카테고리를 모두 표시하고, 카테고리 행 클릭으로 `include_in_statistics`를 전환
* 통계 제외 바텀시트에는 별도 토글 스위치를 두지 않으며, 제외 상태는 회색 처리와 `통계 제외` 텍스트 배지로 표시
* 통계 제외 바텀시트의 행 터치 영역은 모바일 기준 최소 44px 높이를 유지하고 Safe Area 하단 여백을 적용

---

## API 연결

| 기능      | API                   |
| ------- | --------------------- |
| 카테고리 조회 | GET /api/v1/categories       |
| 카테고리 등록 | POST /api/v1/categories      |
| 카테고리 수정 | PUT /api/v1/categories/{id} |

카테고리 삭제 API는 연결하지 않는다. 카테고리 숨김은 `PUT /api/v1/categories/{id}`에 `show=false`를 전달해 처리한다.
카테고리 통계 제외는 `PUT /api/v1/categories/{id}`에 `include_in_statistics=false`를 전달해 처리한다.

# 14. 아이콘 관리 화면

## 화면 설명

내장 아이콘, 이모지, 코드값 기반 아이콘을 등록 및 관리하는 화면이다.
이미지 파일 업로드는 MVP에서 제외한다.

---

## 주요 기능

* 사용자 아이콘 영역의 등록 버튼을 통해 아이콘 등록 화면을 연다
* 아이콘 등록 화면에서는 이름을 검색하고, 서버 Icon Provider Adapter 검색 결과에서 아이콘을 선택한 뒤 등록 버튼으로 등록한다
* 사용 여부 관리
* 목록 조회

---

## 정책 정의

* 아이콘은 내부 아이콘 코드와 Provider 매핑으로 저장한다. 이미지 파일 업로드는 미지원
* 표시 여부가 ON인 아이콘만 선택 화면에 표시
* 아이콘 검색은 서버의 Icon Provider Adapter를 통해 수행하며 현재 기본 Provider는 Lucide Icons다
* 아이콘 선택 및 목록 UI에는 아이콘 그래픽만 표시하고 아이콘 코드/Provider Key 텍스트는 화면에 노출하지 않는다
* 아이콘 등록 UI는 기본 아이콘 목록을 처음부터 노출하지 않고, 사용자가 입력한 이름 검색 결과만 표시한다
* 아이콘 숨김은 신규 선택 화면에서만 제외하며, 기존 계좌/카드/카테고리 표시는 유지
* 아이콘 관리 목록의 표시 여부는 별도 숨김/표시 버튼 없이 아이콘 카드 색상으로 표현한다
* 사용자 등록 아이콘은 아이콘 카드를 클릭해 표시/숨김 상태를 전환한다

---

## API 연결

| 기능     | API         |
| ------ | ----------- |
| 아이콘 조회 | GET /api/v1/icons  |
| 아이콘 등록 | POST /api/v1/icons |
| 아이콘 표시/숨김 변경 | PUT /api/v1/icons/{id} |

---

# 15. 아이콘 선택 팝업/바텀시트

## 화면 설명

계좌, 카드, 카테고리 등록 및 수정 시 공통으로 사용하는 아이콘 선택 팝업/바텀시트이다.

---

## 주요 기능

* 아이콘 목록 조회
* 아이콘 선택 (단일 탭으로 즉시 선택)
* 선택 결과 반환

---

## 정책 정의

* 아이콘은 단일 탭(single tap) 즉시 선택으로 처리한다
* 사용자가 아이콘을 클릭하면 해당 아이콘이 선택된다
* 선택 UI는 별도 페이지 이동보다 팝업/바텀시트 형태를 우선 사용한다
* 선택 완료 후 호출 화면에 즉시 반영하고 팝업을 닫는다
* 선택 결과는 호출 화면에 `icon_id`와 `icon_code`로 반환한다
* `show=true` 아이콘만 선택 목록에 표시한다
* 선택 목록에는 아이콘 이름 텍스트를 표시하지 않고 아이콘 그래픽만 표시한다

---

## 공통 사용 화면

* 계좌 등록
* 카드 등록
* 카테고리 등록

## 주요 기능

* 닉네임 수정
* 캐시 초기화 (클라이언트 로컬 기능. 별도 API 없음. Cache Storage 및 TanStack Query Cache 삭제 처리)
* 로그아웃

* 로그인 상태 관리

## 예외 처리

* 닉네임 수정 실패
* 설정 저장 실패
* 회원 탈퇴 모달: "탈퇴하면 등록한 데이터가 삭제되며 복구할 수 없습니다." 안내 문구 표시 후 확인 문구(`탈퇴`) 입력 필요
* 회원 탈퇴 불가 (미동기화 데이터 존재 시): 탈퇴 요청 전 프론트에서 로컬 IndexedDB sync_queue를 확인한다. pending/failed 항목이 있으면 탈퇴를 차단하고 동기화 완료 안내 메시지를 표시한다
* 회원 탈퇴 성공 시: 로컬 인증 정보, 사용자 캐시, 화면 상태 제거 후 로그인/시작 화면으로 이동
* 로그아웃 실패 시에도 로컬 토큰 삭제 후 로그인 화면 이동

# 17. 앱 설정 화면

## 화면 설명

사용자별 앱 표시 및 동기화 설정을 관리하는 화면이다.

---

## 화면 진입

대시보드 사용자 카드의 설정 아이콘에서 `/settings/app`으로 이동한다.

---

## 주요 기능

* 앱 설정 조회 및 저장
* 앱 테마 선택 및 즉시 미리보기
* 표시 통화 확인
* 자동 동기화 사용 여부 변경

---

## 정책 정의

* 테마는 `cat-pink`, `mint`, `lavender`만 제공한다.
* 기본 테마는 `cat-pink`다.
* 테마 변경은 저장 전에도 현재 화면에 미리 적용한다.
* 저장 실패 시 서버 저장값으로 되돌릴 수 있도록 오류 상태를 표시한다.
* 기존 저장값이 과거 값(`system`, `light`, `dark`)이면 `cat-pink`로 보정해 표시한다.
* 거래내역은 현재 월 단위로 표시하므로 거래내역 표시 개수 설정은 화면에 노출하지 않는다.

---

## 상태 정의

| 상태 | 설명 |
| --- | --- |
| loading | 설정 조회 중 |
| saving | 설정 저장 중 |
| error | 설정 조회 또는 저장 실패 |

---

## API 연결

| 기능 | API |
| --- | --- |
| 설정 조회 | GET /api/v1/settings |
| 설정 저장 | PUT /api/v1/settings |
