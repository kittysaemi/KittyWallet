# 예외처리

## 1. 문서 목적

본 문서는 미냥이 지갑의 화면, API, 데이터 저장, 오프라인 동기화 과정에서 발생할 수 있는 예외 상황과 처리 기준을 정의한다.

예외 처리는 다음 원칙을 따른다.

* 사용자가 현재 상황을 이해할 수 있어야 한다.
* 데이터 손실을 방지해야 한다.
* 중복 요청을 방지해야 한다.
* 복구 가능한 경우 재시도 수단을 제공해야 한다.
* 서버 검증을 최종 기준으로 한다.

---

# 2. 공통 예외 처리 원칙

| 예외      | 처리 기준              |
| ------- | ------------------ |
| 필수값 누락  | 항목별 오류 메시지 표시      |
| 잘못된 입력값 | 저장 차단 및 오류 표시      |
| API 실패  | 오류 메시지 및 재시도 버튼 표시 |
| 네트워크 오류 | 오프라인 상태 안내         |
| 중복 요청   | 요청 중 버튼 비활성화       |
| 인증 만료   | 토큰 재발급 후 재요청       |
| 재인증 실패  | 로그인 화면 이동          |
| 서버 오류   | 공통 오류 메시지 표시       |

# 3. 인증 예외

## 3.1 로그인 실패

| 조건                 | 처리               |
| ------------------ | ---------------- |
| 이메일 없음             | 이메일 입력 안내        |
| 비밀번호 없음            | 비밀번호 입력 안내       |
| 계정 정보 불일치 또는 미가입 이메일 | "가입되어 있지 않은 계정입니다." 메시지 표시 |
| 서버 오류              | 잠시 후 다시 시도 안내    |
| 네트워크 오류            | 네트워크 연결 확인 안내    |

## 3.2 토큰 만료

| 조건               | 처리                     |
| ---------------- | ---------------------- |
| Access Token 만료  | Refresh Token으로 재발급 시도 |
| 재발급 성공           | 기존 요청 재시도              |
| Refresh Token 만료 | 로그인 화면 이동              |
| 재발급 실패           | 사용자 상태 초기화 후 로그인 이동    |

---

## 3.3 로그아웃 예외

| 조건         | 처리             |
| ---------- | -------------- |
| 로그아웃 요청 실패 | 로컬 인증 정보 우선 삭제 |
| 네트워크 오류    | 클라이언트 상태 초기화   |
| 완료         | 로그인 화면 이동      |

---

## 3.4 비밀번호 재설정 요청 예외

| 조건 | 처리 |
| --- | --- |
| 가입 이메일의 메일 발송 실패 | 요청자에게는 기존 성공 응답과 동일하게 처리하고 서버 로그만 남김 |
| 미가입 이메일 | 계정 열거 방지를 위해 성공 응답과 동일하게 처리 |
| reset token 만료 또는 불일치 | 비밀번호 변경 실패 안내, 기존 비밀번호 유지 |
| 새 비밀번호 확인 불일치 | 비밀번호 확인 불일치 안내 |

# 5. 계좌 관리 예외

| 조건             | 처리                    |
| -------------- | --------------------- |
| 계좌명 미입력        | 계좌명 입력 안내             |

# 13. API 예외

| HTTP 상태 | 처리               |
| ------- | ---------------- |
| 400     | 입력값 오류 표시        |
| 401     | 토큰 재발급 또는 로그인 이동 |
| 403     | 접근 권한 없음 안내      |
| 404     | 데이터 없음 안내        |

| 500     | 서버 오류 안내         |
| Timeout | 네트워크 상태 확인 및 재시도 |

# 15. 공통 메시지 기준

| 상황      | 메시지 방향            |
| ------- | ----------------- |
| 저장 성공   | 저장되었습니다.          |
| 수정 성공   | 수정되었습니다.          |
| 삭제 성공   | 삭제되었습니다.          |
| 저장 실패   | 저장 중 오류가 발생했습니다.  |
| 조회 실패   | 데이터를 불러오지 못했습니다.  |
| 네트워크 오류 | 네트워크 연결을 확인해 주세요. |
| 인증 만료   | 다시 로그인해 주세요.      |
| 빈 데이터   | 표시할 데이터가 없습니다.    |

## 영수증 OCR 예외

| 상황 | 처리 |
|---|---|
| 카메라 권한 거부 또는 미지원 | 사진첩 선택 fallback 제공 |

| OCR 결과 불충분 또는 시간 초과 | 직접 입력 전환 가능, 거래 미생성 |

# 16. 처리 우선순위

예외 발생 시 다음 순서로 처리한다.

1. 사용자 입력 검증
2. 인증 상태 검증
3. 네트워크 상태 확인
4. 서버 요청
5. 서버 응답 검증
6. 상태 업데이트
7. 사용자 메시지 표시
