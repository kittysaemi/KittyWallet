# 상태정의

## 1. 문서 목적

본 문서는 미냥이 지갑의 상태 체계를 상태 소유자 기준으로 정의한다.

화면 문서, 프론트 상태관리 문서, PWA 상태 문서, API 오류 처리 문서는 본 문서를 기준으로 상태명을 사용한다.

---

# 2. 상태 소유자 원칙

| 상태 소유자 | 책임 | 대표 저장 위치 |
|---|---|---|
| UI State | 화면 표시 상태 | 컴포넌트 local state 또는 UI store |
| Auth State | 인증 세션 상태 | Auth store, API client |
| Network State | 네트워크 연결 상태 | PWA store |
| PWA Install State | 설치 가능/완료 상태 | PWA store |
| Cache State | 캐시 준비와 사용 가능 여부 | TanStack Query, Cache Storage, PWA store |
| Sync State | 오프라인 Queue 동기화 상태 | IndexedDB, Sync store |
| Update State | 새 버전 감지와 적용 상태 | Service Worker, PWA store |

---

# 3. 상태 우선순위

동시에 여러 상태가 발생하면 다음 순서로 사용자에게 표시한다.

| 우선순위 | 상태 | 표시 기준 |
|---|---|---|
| 1 | expired | 인증 만료로 화면 접근 불가 |
| 2 | update_available | 새 버전 적용 필요 |
| 3 | sync_failed | 미동기화 데이터 처리 실패 |
| 4 | offline | 네트워크 연결 없음 |
| 5 | pending_sync | 동기화 대기 데이터 존재 |
| 6 | error | 현재 화면 요청 실패 |
| 7 | loading | 최초 조회 또는 저장 중 |
| 8 | empty | 표시할 데이터 없음 |

우선순위가 높은 상태가 낮은 상태를 완전히 숨기지는 않는다. 예를 들어 `offline + pending_sync`는 오프라인 배너와 동기화 대기 배지를 함께 표시한다.

---

# 4. UI State

UI State는 단일 화면 또는 컴포넌트가 소유한다.

| 상태 | 전이 조건 | 표시 컴포넌트 | 복구 액션 | 관련 API 오류코드 |
|---|---|---|---|---|
| idle | 화면 진입 전 또는 요청 전 | 기본 화면 | 없음 | 없음 |
| loading | 최초 API 요청 시작 | Skeleton, Loading Spinner | 요청 완료 대기 | 없음 |
| refreshing | 기존 데이터가 있는 상태에서 재조회 | Inline Spinner, Pull-to-refresh 표시 | 요청 완료 대기 | 없음 |
| saving | 저장/수정/삭제 요청 중 | 버튼 loading, 입력 비활성 | 중복 요청 차단 | 없음 |
| empty | 조회 성공 후 표시 데이터 없음 | Empty State | 등록 화면 이동, 필터 초기화 | 없음 |
| error | API 실패 또는 렌더링 실패 | Error State, 재시도 버튼 | 재시도, 이전 화면 이동 | 공통 HTTP 400/500, 도메인 오류 |

UI State는 서버 데이터 자체를 소유하지 않는다. 서버 데이터 상태는 TanStack Query의 Server State를 사용하고, UI State는 표시 상태만 관리한다.

---

# 5. Auth State

Auth State는 인증 세션과 보호 라우트 접근 가능 여부를 소유한다.

| 상태 | 전이 조건 | 표시 컴포넌트 | 복구 액션 | 관련 API 오류코드 |
|---|---|---|---|---|
| unauthenticated | 토큰 없음, 로그아웃 완료 | 로그인 화면 | 로그인 | AUTH_002 |
| authenticated | 로그인 성공, 유효한 Access Token 존재 | 보호 화면 표시 | 없음 | 없음 |
| refreshing_token | Access Token 만료 후 refresh 요청 중 | 전체 화면 차단 없음, 요청 대기 | refresh 완료 대기 | AUTH_004 |
| expired | refresh 실패, 토큰 만료 | 로그인 이동 안내 | 인증 상태 초기화 후 로그인 이동 | AUTH_004, HTTP 401 |
| logout_pending | 로그아웃 요청 중 | 로그아웃 버튼 loading | 로컬 토큰 삭제 | AUTH_006 |

인증 만료는 화면별 `error`로 처리하지 않고 `expired`로 승격한다.

---

# 6. Network State

Network State는 브라우저 네트워크와 API 통신 가능 여부를 소유한다.

| 상태 | 전이 조건 | 표시 컴포넌트 | 복구 액션 | 관련 API 오류코드 |
|---|---|---|---|---|
| online | 브라우저 online 이벤트 또는 health check 성공 | 표시 없음 | 백그라운드 재조회, 동기화 시작 | 없음 |
| offline | 브라우저 offline 이벤트 또는 반복 네트워크 실패 | Offline Banner | 캐시 표시, 오프라인 저장 | ERR_NETWORK, Timeout |
| reconnecting | online 복귀 후 health check 중 | 작은 상태 배지 | health check 완료 대기 | 없음 |

`offline`은 화면의 `error`가 아니다. 조회 캐시가 있으면 Cache State와 함께 화면을 유지한다.

---

# 7. PWA Install State

PWA Install State는 설치 가능 여부와 설치 안내 노출을 소유한다.

| 상태 | 전이 조건 | 표시 컴포넌트 | 복구 액션 | 관련 API 오류코드 |
|---|---|---|---|---|
| unsupported | Service Worker, Manifest, HTTPS 조건 불충족 | 설치 안내 미노출 | 일반 브라우저 모드 유지 | 없음 |
| installable | beforeinstallprompt 발생, 설치 조건 충족 | Install Banner | 설치 실행 또는 닫기 | 없음 |
| installed | standalone 실행 또는 설치 완료 | 설치 배너 숨김 | 없음 | 없음 |
| dismissed | 사용자가 설치 배너 닫음 | 배너 숨김 | 재노출 조건 충족 시 installable | 없음 |

설치 상태는 인증 상태와 독립적이다. 설치 가능 상태여도 인증 만료 시 로그인 화면이 우선한다.

---

# 8. Cache State

Cache State는 캐시 생성, 조회 가능 여부, 캐시 실패를 소유한다.

| 상태 | 전이 조건 | 표시 컴포넌트 | 복구 액션 | 관련 API 오류코드 |
|---|---|---|---|---|
| cache_loading | 앱 시작, 캐시 초기화, precache 진행 | Skeleton, Splash | 캐시 준비 대기 | 없음 |
| cache_ready | 마지막 성공 응답 또는 precache 사용 가능 | 캐시 데이터 표시 가능 | background refetch | 없음 |
| cache_stale | 캐시 데이터가 있지만 최신성 보장 불가 | 캐시 데이터 배지 | 온라인 복구 시 재조회 | 없음 |
| cache_error | 캐시 읽기/쓰기 실패 | Error Toast 또는 부분 오류 | 네트워크 요청 우선, 캐시 재생성 | CACHE_001 또는 내부 오류 |

`cache_error`는 화면 전체 `error`로 승격하지 않는다. 단 캐시와 네트워크가 모두 실패하면 UI State의 `error`를 함께 표시한다.

---

# 9. Sync State

Sync State는 IndexedDB Queue와 서버 동기화 결과를 소유한다.

| 상태 | 전이 조건 | 표시 컴포넌트 | 복구 액션 | 관련 API 오류코드 |
|---|---|---|---|---|
| pending_sync | sync_queue에 waiting 항목 존재 | 동기화 대기 배지 | 온라인 복구 또는 수동 동기화 | 없음 |
| syncing | `POST /api/v1/sync/upload` 실행 중 | 동기화 진행 배지 | 요청 완료 대기 | 없음 |
| synced | Queue가 모두 성공 처리됨 | 완료 Toast 또는 배지 숨김 | 관련 Query 무효화 | 없음 |
| sync_failed | 항목별 실패 또는 전체 실패 | 동기화 실패 배너, 재시도 버튼 | 재시도, 실패 항목 상세 확인 | SYNC_001 |
| sync_paused | 사용자 설정에서 자동 동기화 비활성(`sync_enabled=false`) | 동기화 일시정지 배지 | 설정에서 동기화 활성화 또는 수동 동기화 | 없음 |
| sync_conflict | updated_at 기준 충돌 | 충돌 이력 배지 (내부 기록용) | 클라이언트 우선 자동 반영 후 서버 데이터로 화면 갱신. 사용자 선택 UI 없음 | SYNC_002 |
| duplicate_ignored | 동일 client_temp_id 재전송 감지 | 일반적으로 표시 없음 | 기존 transaction_id 매핑 | SYNC_003 |
| storage_unavailable | IndexedDB 생성 실패 | 오프라인 기능 비활성 안내 Toast | 브라우저 저장소 권한 확인 안내. 오프라인 거래 등록/동기화 기능 비활성 | 없음 |
| withdrawal_blocked | 회원 탈퇴 시도 시 로컬 sync_queue에 pending/failed 항목 존재 | 탈퇴 차단 안내 모달 | 동기화 완료 후 재시도 안내 | USER_003 |

`sync_failed`는 화면 UI State의 `error`가 아니라 Sync State의 오류이다. 화면은 별도 배너나 상태 배지로 표시한다.

---

# 10. Update State

Update State는 Service Worker 새 버전 감지와 적용 가능 여부를 소유한다.

| 상태 | 전이 조건 | 표시 컴포넌트 | 복구 액션 | 관련 API 오류코드 |
|---|---|---|---|---|
| update_available | 새 Service Worker 대기 | 업데이트 배너 | 업데이트 적용 |
| update_blocked | pending_sync 존재 | 업데이트 지연 안내 | 동기화 완료 후 업데이트 | 없음 |
| updating | 새 버전 적용 중 | 업데이트 진행 표시 | 완료 대기 | 없음 |
| update_completed | 새 버전 적용 완료 | 새로고침 안내 | 앱 reload | 없음 |
| update_failed | 업데이트 적용 실패 | 재시도 버튼 | 재시도 또는 다음 실행 때 적용 | 없음 |

`pending_sync`가 있으면 `update_available`은 `update_blocked`로 표시한다.

---

# 11. 상태와 HTTP/API 오류 매핑

| HTTP/API 오류 | 소유 상태 | 화면 처리 |
|---|---|---|
| HTTP 400, VALIDATION_001 | UI State error | 필드 오류 또는 요청 오류 표시 |
| HTTP 401, AUTH_004 | Auth State expired | 토큰 초기화 후 로그인 이동 |
| HTTP 403, AUTH_009 | UI State error | 접근 불가 안내 |
| HTTP 404 | UI State empty 또는 error | 목록은 empty, 상세는 error |
| HTTP 409, SYNC_002 | Sync State sync_conflict | 클라이언트 우선 자동 반영. 사용자 선택 UI 없음. SYNC_HISTORY에 CONFLICT 기록 |
| SYNC_001 | Sync State sync_failed | 동기화 실패 배너 |
| SYNC_003 | Sync State duplicate_ignored | 기존 서버 ID 매핑 후 Queue 정리 |
| SETTING_001 | UI State error | 설정 폼 오류 표시 |
| Timeout, Network Error | Network State offline 또는 UI State error | 네트워크 확인 후 재시도 |

# 12. 화면별 상태 사용 기준

| 화면 | 필수 상태 | 선택 상태 |
|---|---|---|
| 로그인 | loading, error, offline | expired |
| 회원가입 | loading, error, offline | 없음 |
| 대시보드 | loading, refreshing, empty, error, offline | pending_sync, syncing, sync_failed |
| 거래 등록/수정 | saving, error, offline | pending_sync, syncing |
| 거래 검색 | loading, refreshing, empty, error | offline, cache_ready |
| 통계 | loading, empty, error | cache_ready, cache_stale |
| 설정 | loading, saving, error | sync_failed |
| PWA 설치 안내 | installable, installed, dismissed, unsupported | 없음 |

영수증 OCR과 텍스트 분석은 신규 거래 등록의 local UI 상태로만 관리한다. `source_select`, `file_selecting`, `text_input`, `analyzing`, `analysis_error`, `analysis_unavailable_offline` 상태를 사용하며 이미지·OCR 원문을 전역 상태나 브라우저 영구 저장소에 두지 않는다. 카메라 입력은 모바일 파일 입력의 `capture` 힌트를 사용하므로 별도의 웹캠 권한 상태를 관리하지 않는다.

# 13. 구현 기준

| 상태 영역 | 권장 구현 |
|---|---|
| UI State | 컴포넌트 local state, 필요한 경우 UI store |
| Auth State | Auth store + API client interceptor |
| Network State | PWA store + browser online/offline + health check |
| PWA Install State | PWA store |
| Cache State | TanStack Query + Cache Storage 상태 |
| Sync State | IndexedDB sync_queue + Sync store |
| Update State | Service Worker 이벤트 + PWA store |

Form 입력값은 상태정의 대상이 아니며 React Hook Form 등 폼 전용 상태로 관리한다.

---

# 14. 금지 사항

* 화면별로 임의 상태명을 새로 만들지 않는다.
* `expired`를 단순 `error`로 처리하지 않는다.
* `sync_failed`를 화면 전체 장애로 처리하지 않는다.
* `cache_error`만으로 IndexedDB를 삭제하지 않는다.
* 여러 store가 같은 상태를 중복 소유하지 않는다.
