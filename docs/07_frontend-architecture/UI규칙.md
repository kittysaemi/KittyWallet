# UI 규칙

## 1. 문서 목적

본 문서는 미냥이 지갑 프론트엔드 UI 구성 규칙을 정의한다.

---

# 2. UI 설계 목표

- 모바일 최적화
- 단순한 입력 흐름
- 빠른 거래 등록
- 공통 컴포넌트 재사용
- 접근성 확보

---

# 3. 공통 UI 규칙

| 항목 | 기준 |
|---|---|
| 기본 단위 | 8px |
| 최소 터치 영역 | 44px |
| 버튼 높이 | 44px 이상 |
| 입력창 높이 | 44px 이상 |

---

# 4. 색상 규칙

| 구분 | 목적 |
|---|---|
| Primary | 주요 액션 |
| Danger | 삭제 |
| Success | 완료 |
| Warning | 경고 |

---

# 5. 타이포그래피 규칙

| 항목 | 기준 |
|---|---|
| 기본 폰트 | Pretendard |
| 본문 최소 크기 | 14px |
| 제목 최소 크기 | 18px |
| 줄간격 | 1.4 이상 |

---

# 6. 입력 UI 규칙

- 거래 등록 화면은 단일 흐름 유지
- 입력 단계 최소화
- 숫자 입력은 키패드 사용
- 필수값은 명확히 표시

---

# 7. 목록 UI 규칙

- 스크롤 최적화 적용
- Infinite Scroll 가능 구조
- Skeleton UI 지원
- Empty 상태 표시

---

# 8. 모달 규칙

| 항목 | 기준 |
|---|---|
| 닫기 버튼 | 필수 |
| 배경 클릭 종료 | 선택 |
| ESC 종료 | 지원 |

---

# 9. 로딩 처리 규칙

- 전체 화면 로딩 최소화
- 부분 로딩 우선 사용
- 요청 실패 시 재시도 UI 제공

---

# 10. 모바일 대응 규칙

- Safe Area 대응
- 하단 네비게이션 고려
- 키보드 올라옴 대응
- 터치 우선 UI 구성

---

# 12. 접근성 규칙

- aria-label 사용
- 키보드 접근 가능
- 명확한 포커스 표시
