# 상태관리

## 1. 문서 목적

본 문서는 미냥이 지갑 프론트엔드의 상태 관리 구조를 정의한다.

상태명, 상태 소유자, 전이 조건, 표시 컴포넌트, 복구 액션의 canonical 기준은 `docs/03_screen-spec/상태정의.md`를 따른다.

---

# 2. 상태 관리 목표

- 서버 상태와 UI 상태 분리
- 불필요한 전역 상태 최소화
- API 캐싱 지원
- 오프라인 저장 지원
- 동기화 충돌 최소화

---

# 3. 상태 분류

| 구분 | 설명 |
|---|---|
| Server State | API 데이터 |
| UI State | Modal, Loading |
| Form State | 입력 데이터 |
| Sync State | 동기화 상태 |
| Auth State | 로그인 상태 |
| Network State | 온라인/오프라인 상태 |
| PWA State | 설치, 캐시, 업데이트 상태 |

---

# 4. 서버 상태 관리

## 사용 기술

- TanStack Query

---

## 관리 대상

- 거래 목록
- 계좌 목록
- 카드 목록
- 통계 데이터
- 사용자 정보

---

## 캐싱 정책

| 항목 | 기준 |
|---|---|
| staleTime | 화면별 설정 |
| refetchOnWindowFocus | 비활성 |
| retry | 1~3회 |
| background refetch | 사용 |

## 저장 대상

- 최근 조회 데이터

---

# 7. 동기화 상태 관리

| 상태 | 설명 |
|---|---|
| pending_sync | 동기화 대기 |
| syncing | 동기화 중 |
| synced | 동기화 완료 |
| sync_failed | 동기화 실패 |
| sync_conflict | 충돌 발생 |
| duplicate_ignored | 중복 요청 무시 |

---

# 8. 상태 관리 규칙

- 서버 데이터는 Query 사용
- UI 상태만 Zustand 사용
- Form 상태 전역 저장 금지
- Query Cache 직접 수정 최소화
