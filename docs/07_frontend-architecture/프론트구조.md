# 프론트 구조

## 1. 문서 목적

본 문서는 미냥이 지갑 프론트엔드 프로젝트의 전체 구조와 계층 설계 기준을 정의한다.

본 구조는 다음 목표를 기준으로 설계한다.

- 기능 확장 시 최소 수정
- API 변경 영향 최소화
- 기능 단위 독립성 확보
- 오프라인 구조 대응
- 유지보수성 향상
- 공통 UI 재사용성 확보

---

# 2. 기술 스택

| 항목 | 기술 |
|---|---|
| Framework | React |
| Language | TypeScript |
| Build Tool | Vite |
| Routing | React Router |
| Server State | TanStack Query |
| Local State | Zustand |
| Storage | IndexedDB |
| PWA | vite-plugin-pwa |

---

# 3. 전체 폴더 구조

```text
src/
  app/
  pages/
  features/
  entities/
  shared/
```

---

# 4. 계층 구조 정의

## 4.1 app

앱 전체 초기화 및 공통 설정 관리.

```text
app/
  router/
  providers/
  layouts/
```

### 역할

- Router 설정
- Provider 등록
- 공통 Layout 구성
- 앱 초기화 처리

---

## 4.2 pages

화면 단위 구성.

```text
pages/
  login/
  dashboard/
  transaction/
  account/
  card/
  statistics/
```

### 역할

- 화면 조립
- feature 연결
- layout 적용
- 페이지 단위 데이터 구성

---

## 4.3 features

사용자 액션 중심 기능 모듈.

```text
features/
  transaction/
    create/
    update/
    delete/
```

### 역할

- 기능 처리
- API Mutation
- Form 처리
- Validation 처리

---

## 4.4 entities

핵심 도메인 데이터 단위.

```text
entities/
  transaction/
  account/
  card/
  category/
```

### 역할

- API
- Model
- Types
- Domain UI

---

## 4.5 shared

공통 모듈.

```text
shared/
  api/
  ui/
  hooks/
  utils/
  storage/
```

### 역할

- 공통 UI
- 공통 API
- 공통 Hook
- 공통 Utility
- Storage 처리

---

# 5. 의존성 규칙

## 허용 방향

```text
pages → features → entities → shared
```

---

## 금지 규칙

- shared가 entities 의존 금지
- entities가 pages 의존 금지
- feature 간 직접 의존 최소화
