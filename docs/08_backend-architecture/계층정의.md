# 계층 정의

## 1. 문서 목적

본 문서는 미냥이 지갑 백엔드의 계층 구조와 각 계층의 책임, 의존성 방향, 구현 기준을 정의한다.

---

# 2. 계층 구조

```text
Presentation Layer
  ↓
Application Layer
  ↓
Domain Layer
  ↓
Infrastructure Layer
```

NestJS 구조에서는 각 계층을 모듈 내부 폴더로 분리한다.

---

# 3. Presentation Layer

## 3.1 역할

Presentation Layer는 HTTP 요청과 응답을 담당한다.

## 3.2 포함 대상

* Controller
* Request DTO
* Response DTO
* Query DTO
* Param DTO

## 3.3 책임

| 책임 | 설명 |
|---|---|
| 요청 수신 | API 요청을 받는다 |
| 인증 사용자 전달 | Guard에서 확인된 user_id를 UseCase로 전달한다 |
| DTO 검증 | 입력값 형식 검증을 수행한다 |
| 응답 반환 | UseCase 결과를 공통 응답 형태로 반환한다 |

## 3.4 금지 사항

* 비즈니스 정책 처리 금지
* Prisma 직접 호출 금지
* 잔액 계산 금지
* 트랜잭션 처리 금지
* Entity 직접 반환 금지

---

# 4. Application Layer

## 4.1 역할

Application Layer는 사용자의 요청을 실제 기능 단위로 처리하는 계층이다.

## 4.2 포함 대상

* UseCase
* Application Service
* Command
* Query

## 4.3 책임

| 책임 | 설명 |
|---|---|
| 기능 흐름 제어 | 등록, 수정, 삭제 등 처리 순서를 제어한다 |
| 정책 호출 | Domain Policy를 사용해 업무 규칙을 검증한다 |
| Repository 호출 | 데이터 접근을 Repository Interface를 통해 수행한다 |
| Transaction 제어 | 여러 데이터 변경을 하나의 DB Transaction으로 묶는다 |
| DTO 변환 | 결과를 Response DTO로 변환한다 |

## 4.4 금지 사항

* HTTP Request 객체 직접 참조 금지
* Prisma Client 직접 참조 금지
* 환경 변수 직접 참조 금지

---

# 5. Domain Layer

## 5.1 역할

Domain Layer는 미냥이 지갑의 핵심 업무 규칙을 표현한다.

## 5.2 포함 대상

* Domain Model
* Policy
* Value Object
* Domain Constant
* Domain Service

## 5.3 주요 정책 예시

| 정책 | 설명 |
|---|---|
| TransactionAmountPolicy | 금액은 0보다 커야 한다 |
| TransactionDatePolicy | 미래 날짜 등록 불가 |
| AccountBalancePolicy | 수입/지출/삭제 시 잔액 계산 |
| SoftDeletePolicy | deleted_yn 또는 use_yn 기준 처리 |
| UserOwnershipPolicy | user_id 기준 데이터 접근 제한 |

## 5.4 금지 사항

* DB 접근 금지
* HTTP 응답 생성 금지
* 외부 라이브러리 강결합 금지

---

# 6. Infrastructure Layer

## 6.1 역할

Infrastructure Layer는 실제 기술 구현을 담당한다.

## 6.2 포함 대상

* Prisma Repository 구현체
* Prisma Service
* JWT Service Wrapper
* Password Hasher
* File Storage 구현체
* 외부 API Client

## 6.3 책임

| 책임 | 설명 |
|---|---|
| DB 접근 | Prisma를 통한 데이터 조회 및 저장 |
| 외부 기술 처리 | JWT, 암호화, 파일 저장 등 |
| Interface 구현 | Application이 사용하는 인터페이스 구현 |

## 6.4 금지 사항

* Controller 직접 참조 금지
* API 응답 형식 생성 금지
* 비즈니스 흐름 제어 금지

---

# 7. Common Layer

Common Layer는 모든 계층에서 사용할 수 있는 공통 요소를 제공한다.

| 구성 | 설명 |
|---|---|
| Exception | 공통 예외 클래스 |
| Filter | 전역 예외 처리 |
| Interceptor | 응답 변환 및 로깅 |
| Guard | 인증 및 권한 확인 |
| Decorator | CurrentUser 등 편의 기능 |
| Response | 공통 응답 Wrapper |
| Utility | 공통 유틸 함수 |

---

# 8. 계층별 데이터 전달 기준

```text
Controller → Request DTO
UseCase → Command / Query
Repository → Domain Model 또는 Persistence Model
Controller ← Response DTO
```

Prisma Model은 Infrastructure 내부에서만 사용하고, 외부 계층으로 직접 전달하지 않는다.

---

# 9. 최종 기준

* Controller는 얇게 유지한다.
* UseCase는 기능 흐름을 명확하게 가진다.
* Domain은 정책과 계산 규칙을 담당한다.
* Infrastructure는 기술 구현만 담당한다.
* 계층 간 의존성 방향은 반드시 단방향으로 유지한다.
