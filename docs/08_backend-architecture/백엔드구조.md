# 백엔드 구조

## 1. 문서 목적

본 문서는 미냥이 지갑 백엔드 프로젝트의 폴더 구조와 각 디렉터리의 역할을 정의한다.

백엔드 구조는 기능 확장, 유지보수, 테스트, AI 협업 개발이 가능한 형태를 기준으로 한다.

---

---

# 3. src 구조

```text
src
 ├── main.ts
 ├── app.module.ts
 ├── config
 ├── common
 ├── database
 ├── modules
 └── infra
```

| 경로 | 역할 |
|---|---|
| main.ts | 서버 부트스트랩, 전역 설정 적용 |
| app.module.ts | 루트 모듈 구성 |
| config | 환경 변수 및 설정 관리 |
| common | 공통 응답, 예외, 데코레이터, 유틸 |
| database | Prisma 연결 및 DB 공통 처리 |
| modules | 기능별 업무 모듈 |
| infra | 외부 기술 구현체, 토큰, 암호화, 파일 처리 |

# 5. 모듈 내부 구조

```text

 ├── transaction.module.ts
 ├── presentation
 │   ├── transaction.controller.ts
 │   └── dto
 ├── application
 │   ├── use-cases
 │   └── services
 ├── domain
 │   ├── models
 │   ├── policies
 │   └── constants

     └── repositories
```

| 영역 | 역할 |
|---|---|
| presentation | Controller, Request DTO, Response DTO |
| application | UseCase, Application Service |
| domain | 정책, 계산 규칙, 도메인 상수 |
| infrastructure | Repository 구현체, Prisma 접근 |

# 10. 구조 적용 원칙

* 기능은 modules 하위에 위치한다.
* 공통 처리는 common에 위치한다.
* Prisma 직접 접근은 infrastructure/repositories 내부로 제한한다.
* Controller는 UseCase만 호출한다.
* UseCase는 Repository Interface에 의존한다.
* DTO는 presentation 하위에 위치한다.
* 도메인 정책은 domain/policies 하위에 위치한다.

---

# 11. 금지 사항

* 모듈 간 내부 파일 직접 참조 금지
* Controller에서 Prisma 직접 접근 금지
* DTO 없이 Request Body 직접 사용 금지
* Response에 Prisma Model 직접 반환 금지
* 환경 변수를 서비스 코드에서 직접 참조 금지
* 공통 예외 형식 외 임의 오류 응답 금지
