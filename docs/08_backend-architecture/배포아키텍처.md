# 배포 아키텍처

## 1. 문서 목적

본 문서는 미냥이 지갑 백엔드의 배포 구조와 실행 환경, Docker 및 Nginx 기준을 정의한다.

---

# 2. 기본 배포 구조

```text
Client
  ↓
Nginx
  ↓
NestJS Backend
  ↓
PostgreSQL
```

---

# 3. 구성 요소

| 구성 | 역할 |
|---|---|
| Nginx | Reverse Proxy, 정적 파일 제공, API Proxy |
| NestJS Backend | REST API 서버 |
| PostgreSQL | 서비스 데이터 저장 |
| Docker | 실행 환경 표준화 |
| .env | 환경별 설정 관리 |

---

# 4. Docker 구성

```text
docker-compose.yml
 ├── backend
 ├── postgres
 └── nginx
```

## backend

* NestJS API 서버 실행
* 환경 변수 기반 설정
* Prisma Client 사용

## postgres

* PostgreSQL 데이터베이스
* volume 기반 데이터 보존

## nginx

* API Reverse Proxy
* HTTPS 적용 가능 구조

---

# 5. 환경 구분

| 환경 | 설명 |
|---|---|
| local | 개발자 로컬 실행 |
| dev | 개발 서버 |
| staging | 배포 전 검증 |
| prod | 운영 환경 |

---

# 6. 환경 변수 기준

| 변수 | 설명 |
|---|---|
| NODE_ENV | 실행 환경 |
| PORT | 서버 포트 |
| DATABASE_URL | PostgreSQL 연결 문자열 |
| JWT_ACCESS_SECRET | Access Token Secret |
| JWT_REFRESH_SECRET | Refresh Token Secret |
| ACCESS_TOKEN_EXPIRES_IN | Access Token 만료 시간 |
| REFRESH_TOKEN_EXPIRES_IN | Refresh Token 만료 시간 |
| CORS_ORIGIN | 허용 Origin |

---

# 7. 배포 흐름

```text
코드 Merge
 → Docker Image Build
 → 환경 변수 적용
 → DB Migration 실행
 → Backend Container 실행
 → Health Check
 → Nginx Proxy 연결
```

---

# 8. DB Migration 기준

* Prisma Migration을 사용한다.
* 운영 DB에는 임의 수동 컬럼 추가를 금지한다.
* Migration 전 백업을 권장한다.
* 배포 전 staging 환경에서 Migration을 검증한다.

---

# 9. Health Check

| API | 설명 |
|---|---|
| GET /health | 서버 상태 확인 |
| GET /health/db | DB 연결 상태 확인 |

---

# 10. 운영 기준

* 운영 환경 Secret은 소스에 포함하지 않는다.
* CORS는 허용 도메인만 등록한다.
* DB Port는 외부 노출을 최소화한다.
* Nginx에서 HTTPS 적용을 고려한다.
* 서버 로그는 민감정보를 제외한다.

---

# 11. 최종 기준

배포 구조는 로컬 개발, 테스트, 운영 환경의 차이를 최소화하고 Docker 기반으로 반복 가능한 실행 환경을 제공해야 한다.
