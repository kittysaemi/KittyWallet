# 백엔드 예외 처리

## 1. 문서 목적

본 문서는 미냥이 지갑 백엔드에서 발생하는 예외 상황과 공통 오류 응답 구조, 처리 기준을 정의한다.

---

# 2. 예외 처리 원칙

| 원칙 | 설명 |
|---|---|
| 일관성 | 모든 API 오류는 동일한 구조로 반환 |
| 명확성 | 사용자가 이해 가능한 메시지 제공 |
| 보안 | 민감한 내부 오류 정보 노출 금지 |
| 복구 가능성 | 재시도 가능한 오류는 명확히 구분 |
| 추적 가능성 | 서버 로그에는 추적 가능한 정보 기록 |

---

# 3. 오류 응답 구조

```json
{
  "success": false,
  "error": {
    "code": "TRANSACTION_AMOUNT_INVALID",
    "message": "거래 금액은 0보다 커야 합니다.",
    "status": 400
  }
}
```

---

# 4. HTTP 상태 코드 기준

| 상태 코드 | 사용 기준 |
|---|---|
| 400 | 입력값 오류, 정책 위반 |
| 401 | 인증 실패, 토큰 만료 |
| 403 | 접근 권한 없음 |
| 404 | 데이터 없음 |
| 409 | 중복 데이터, 충돌 발생 |
| 500 | 서버 내부 오류 |

---

# 5. 공통 오류 코드

| 코드 | 상태 | 설명 |
|---|---|---|
| VALIDATION_ERROR | 400 | 입력값 검증 실패 |
| UNAUTHORIZED | 401 | 인증 필요 |
| TOKEN_EXPIRED | 401 | Access Token 만료 |
| FORBIDDEN | 403 | 권한 없음 |
| NOT_FOUND | 404 | 리소스 없음 |
| CONFLICT | 409 | 중복 또는 충돌 |
| INTERNAL_SERVER_ERROR | 500 | 서버 오류 |

---

# 6. 업무 오류 코드

| 코드 | 상태 | 설명 |
|---|---|---|
| EMAIL_ALREADY_EXISTS | 409 | 이메일 중복 |
| LOGIN_FAILED | 401 | 로그인 실패 |
| TRANSACTION_AMOUNT_INVALID | 400 | 금액 0 이하 입력 |
| TRANSACTION_DATE_INVALID | 400 | 미래 날짜 입력 |
| CATEGORY_REQUIRED | 400 | 카테고리 미선택 |
| ACCOUNT_NOT_FOUND | 404 | 계좌 없음 |
| CARD_NOT_FOUND | 404 | 카드 없음 |
| CATEGORY_NOT_FOUND | 404 | 카테고리 없음 |
| INACTIVE_WALLET | 400 | 비활성 계좌/카드 선택 |
| DELETED_TRANSACTION | 400 | 삭제된 거래 수정 시도 |
| SYNC_CONFLICT | 409 | 동기화 충돌 |

---

# 7. Global Exception Filter

모든 예외는 Global Exception Filter에서 공통 응답 구조로 변환한다.

처리 대상:

* HttpException
* Validation Error
* Prisma Error
* Business Exception
* Unknown Error

---

# 8. Prisma 오류 처리

| Prisma 오류 | 변환 기준 |
|---|---|
| Unique Constraint | 409 Conflict |
| Foreign Key Constraint | 400 Bad Request |
| Record Not Found | 404 Not Found |
| DB Connection Error | 500 Internal Server Error |

---

# 9. 로그 기준

* 4xx 오류는 warn 수준으로 기록한다.
* 5xx 오류는 error 수준으로 기록한다.
* 비밀번호, 토큰, 개인정보는 로그에 남기지 않는다.
* request_id 또는 trace_id를 포함할 수 있다.

---

# 10. 금지 사항

* stack trace를 클라이언트에 반환 금지
* Prisma 원본 오류 메시지 직접 반환 금지
* 모듈별로 다른 오류 응답 구조 사용 금지
* 민감정보 포함 메시지 반환 금지
