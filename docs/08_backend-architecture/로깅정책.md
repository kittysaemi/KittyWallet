# 로깅 정책

## 1. 문서 목적

본 문서는 미냥이 지갑 백엔드의 로그 기록 대상, 로그 레벨, 민감정보 제외 기준을 정의한다.

---

# 2. 로깅 목표

* API 오류 원인 추적
* 인증 및 주요 데이터 변경 이력 확인
* 동기화 실패 원인 확인
* 운영 장애 대응
* 민감정보 보호

---

# 3. 로그 레벨

| 레벨 | 사용 기준 |
|---|---|
| debug | 개발 환경 상세 확인 |
| info | 정상 요청, 주요 처리 완료 |
| warn | 복구 가능한 오류, 4xx 오류 |
| error | 서버 오류, DB 오류, 5xx 오류 |

---

# 4. 로그 대상

| 대상 | 기록 내용 |
|---|---|
| API Request | method, path, user_id, status, duration |
| API Error | error_code, status, path, user_id |
| Auth | 로그인 성공/실패, 토큰 재발급 실패 |
| Transaction | 거래 등록/수정/삭제 결과 |
| Sync | 동기화 성공/실패, 실패 항목 수 |
| DB Error | DB 오류 종류, 발생 위치 |

---

# 5. 민감정보 제외 기준

로그에 기록하지 않는 정보:

* 비밀번호
* Access Token
* Refresh Token
* 카드번호 등 금융 민감정보
* 인증 헤더 전체 값
* 개인정보 전체 원문

---

# 6. 요청 로그 형식

```json
{
  "level": "info",
  "method": "POST",
  "path": "/transactions",
  "status": 201,
  "userId": "user_id",
  "durationMs": 35
}
```

---

# 7. 오류 로그 형식

```json
{
  "level": "error",
  "code": "INTERNAL_SERVER_ERROR",
  "path": "/transactions",
  "userId": "user_id",
  "message": "Unexpected server error"
}
```

---

# 8. 환경별 기준

| 환경 | 기준 |
|---|---|
| local | debug 허용 |
| dev | debug/info 허용 |
| staging | info/warn/error |
| prod | info/warn/error, 민감정보 제외 필수 |

---

# 9. 금지 사항

* 비밀번호 로그 출력 금지
* 토큰 원문 로그 출력 금지
* Request Body 전체 무조건 출력 금지
* DB 연결 정보 출력 금지
* 운영 환경에서 stack trace 전체 응답 금지
