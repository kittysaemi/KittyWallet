# 문서 작성 규칙

## 최신 기준 유지

문서 본문에는 최신 기준만 유지한다.

변경된 정책, API, ERD, 상태 정의가 기존 내용과 충돌할 경우 본문을 최신 기준으로 수정한다.

기존 기준을 보존해야 할 필요가 있을 경우 본문에 함께 두지 않고 `[DEPRECATED]` 섹션 또는 변경 이력으로 분리한다.

폐기된 기준에는 다음 정보를 함께 남긴다.

| 항목 | 설명 |
|---|---|
| 폐기 기준 | 더 이상 사용하지 않는 내용 |
| 폐기 사유 | 변경 또는 폐기 이유 |
| 대체 기준 | 현재 적용할 최신 기준 |
| 적용 종료일 | 해당 기준을 더 이상 사용하지 않는 날짜 |

오탈자, 표현 정리, 중복 문장 제거처럼 의미가 바뀌지 않는 수정은 변경 이력 없이 바로 정리할 수 있다.

# 문서 폴더 규칙

현재 `docs/00_project/문서인덱스.md`의 분류 체계를 공식 문서 구조로 사용한다.

문서는 아래 구조에 맞춰 작성한다.

```text
docs/

00_project/
01_service-planning/
02_information-architecture/
03_screen-spec/
04_business-policy/
05_data-design/
06_api-spec/
07_frontend-architecture/
08_backend-architecture/
09_pwa/
10_design-system/
11_testing/
12_deployment/
```

신규 폴더가 필요할 경우 먼저 `문서인덱스.md`에 분류 목적과 참조 우선순위를 반영한 뒤 추가한다.

---

# 문서 참조 우선순위

AI와 개발자의 문서 참조 우선순위는 `docs/00_project/문서인덱스.md`를 따른다.

본 문서는 별도의 상세 참조 순서를 중복 정의하지 않는다.
