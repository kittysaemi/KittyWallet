# 코드 규칙

## 문서 목적

본 문서는 미냥이 지갑 프로젝트의 코드 작성 기준을 정의한다.

모든 개발자는 동일한 구조와 패턴을 사용하여 기능을 구현해야 하며, AI(Codex, Claude Code, Copilot) 또한 본 규칙을 준수해야 한다.

---

# 개발 원칙

## 구조 우선

기능 구현보다 구조 일관성을 우선한다.

임시 구현보다 유지보수 가능한 구조를 선택한다.

---

## 중복 금지

동일한 로직을 여러 곳에 구현하지 않는다.

공통 로직은 shared 계층으로 이동한다.

---

## 하드코딩 금지

다음 항목은 하드코딩하지 않는다.

* API URL
* 오류 메시지
* 상태 코드
* 카테고리 값
* 거래 유형 값

상수 또는 타입으로 관리한다.

---

# TypeScript 규칙

## any 사용 금지

잘못된 예

```ts
const data: any = response.data;
```

올바른 예

```ts
const data: TransactionResponse = response.data;
```
