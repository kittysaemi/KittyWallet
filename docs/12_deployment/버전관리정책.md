# 버전관리정책.md

# 1. 문서 목적

본 문서는 미냥이 지갑 서비스의 버전 관리 기준을 정의한다.

본 정책은 다음 목적을 가진다.

* 릴리즈 이력 관리
* 배포 기준 통일
* 롤백 기준 제공
* DB Migration 관리
* PWA Cache 관리
* AI 자동화 기준 제공

---

# 2. 적용 범위

본 정책은 다음 대상에 적용한다.

| 구분        | 대상             |
| --------- | -------------- |
| Frontend  | PWA            |
| Backend   | API            |
| Database  | PostgreSQL     |
| Migration | Prisma         |
| Docker    | Image          |
| PWA       | Service Worker |
| Offline   | IndexedDB      |

---

# 3. 버전 체계

SemVer 적용

```text
MAJOR.MINOR.PATCH
```

예시

```text
1.0.0
1.1.0
1.1.1
2.0.0
```

---

# 4. 버전 증가 규칙

## Major

구조 변경

예시

```text
1.0.0
↓
2.0.0
```

적용

* DB 구조 변경
* API Breaking Change
* 인증 구조 변경
* Sync 구조 변경

---

## Minor

기능 추가

예시

```text
1.0.0
↓
1.1.0
```

적용

* 신규 화면
* 신규 API
* 신규 통계
* 신규 설정

---

## Patch

버그 수정

예시

```text
1.1.0
↓
1.1.1
```

적용

* UI 수정
* API 버그 수정
* Hotfix

---

# 5. Git 브랜치 전략

## 기본 구조

```text
main
develop
feature/*
hotfix/*
release/*
```

---

## main

운영 버전만 존재

배포 가능 브랜치

---

## develop

다음 버전 개발 브랜치

---

## feature

기능 개발

예시

```text
feature/transaction-create
feature/statistics-monthly
```

---

## release

배포 준비

예시

```text
release/1.1.0
release/1.2.0
```

---

## hotfix

긴급 수정

예시

```text
hotfix/login-fail
hotfix/sync-error
```

---
