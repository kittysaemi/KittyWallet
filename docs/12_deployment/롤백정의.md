# 롤백정의.md

# 1. 문서 목적

본 문서는 미냥이 지갑 서비스 운영 배포 실패 또는 운영 장애 발생 시 안정적으로 이전 버전으로 복구하기 위한 표준 롤백 절차를 정의한다.

본 문서는 다음 목적을 가진다.

* 서비스 중단 시간 최소화
* 데이터 손실 방지
* 운영자 대응 표준화
* AI 기반 운영 자동화 지원
* 장애 발생 시 신속한 복구 수행

---

# 2. 적용 범위

본 문서는 다음 구성 요소에 적용한다.

| 구분            | 대상         |
| ------------- | ---------- |
| Frontend      | PWA        |
| Backend       | API Server |
| Database      | PostgreSQL |
| Reverse Proxy | Nginx      |
| Storage       | IndexedDB  |
| Sync          | Sync Queue |
| Container     | Docker     |

---

# 3. 롤백 등급

## R1 - 즉시 롤백

5분 이내 롤백 수행

### 대상

* 로그인 불가
* 전체 서비스 장애
* DB Migration 실패
* API 응답 불가
* 데이터 손실 가능성

---

## R2 - 계획 롤백

30분 이내 롤백 수행

### 대상

* 일부 기능 장애
* Sync 장애
* 통계 오류
* UI 치명적 오류

---

## R3 - Hotfix 우선

롤백 대신 수정 배포

### 대상

* 단순 UI 오류
* 문구 오류
* 비핵심 기능 오류

---
