# 배포정책.md

# 1. 문서 목적

본 문서는 미냥이 지갑 서비스의 운영 배포 기준을 정의한다.

배포 정책은 서비스 안정성 확보, 장애 최소화, 운영 환경 일관성 유지, AI 기반 자동 배포 지원을 목적으로 한다.

---

# 2. 적용 범위

* Frontend (PWA)
* Backend API
* Database
* Docker Container
* Nginx
* Service Worker
* Sync 기능

---

# 3. 배포 원칙

## 3.1 브랜치 정책

운영 배포는 main 브랜치 기준으로 수행한다.

```text
feature/*
develop
main
```

### 허용

* feature → develop
* develop → main

### 금지

* feature → main 직접 배포

---

## 3.2 배포 승인 조건

모든 조건 충족 시 배포 가능

| 항목       | 조건 |
| -------- | -- |
| QA       | 완료 |
| 테스트 케이스  | 통과 |
| E2E      | 통과 |
| 회귀 테스트   | 통과 |
| 운영 체크리스트 | 완료 |

---

## 3.3 배포 금지 조건

* Critical Bug 존재
* 로그인 실패
* 거래 저장 오류
* Sync 오류
* DB Migration 검증 미완료
* Rollback 검증 미완료

---

# 7. AI 자동화 기준

AI는 다음 작업 수행 가능

* Release 생성
* Docker Build
* 배포 스크립트 실행
* Health Check 수행

AI는 다음 작업 수행 금지

* Production DB 직접 수정
* 운영 데이터 삭제
* 승인 없는 배포
