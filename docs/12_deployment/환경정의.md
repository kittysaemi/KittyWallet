---

## 인증/메일

```env
PASSWORD_RESET_URL=
SMTP_HOST=
SMTP_PORT=
SMTP_SECURE=
SMTP_USER=
SMTP_PASSWORD=
SMTP_FROM=
```

- `PASSWORD_RESET_URL`은 비밀번호 재설정 이메일에 포함할 프론트엔드 URL이다.
- 운영 환경에서 `FRONTEND_URL`은 CORS origin으로 사용하므로 경로 없이 `https://kittysaemi.com`으로 설정한다.
- 운영 환경에서 `API_URL`은 `https://kittysaemi.com/kittywallet/api/v1`을 사용한다.
- 운영 환경에서 `PASSWORD_RESET_URL`은 `https://kittysaemi.com/kittywallet/reset-password`를 사용한다.
- SMTP 비밀값은 Git에 저장하지 않고 환경 변수, Secret Store, CI/CD Secret으로 관리한다.

# 6. KittyWallet 환경 파일 분리

## 개발 환경

개발 환경은 다음 파일을 기준으로 한다.

```text
.env.development
```

저장소에는 실제 값 대신 예시 파일만 포함한다.

```text
.env.development.example
```

## 운영 환경

운영 환경은 서버의 다음 파일을 기준으로 한다.

```text
.env.production
```

저장소에는 실제 값 대신 예시 파일만 포함한다.

```text
.env.production.example
```

## 운영 Compose 실행

개발 Docker Compose에서 `.env.development`를 사용할 때는 다음 명령을 사용한다.

```bash
KITTYWALLET_ENV_FILE=.env.development docker compose up -d
```

운영 서버에서는 다음 명령으로 환경변수 파일을 명시한다.

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml up -d
```

## Secret 관리

금지:

```text
.env.production Git 커밋
JWT_SECRET 평문 커밋
JWT_REFRESH_SECRET 평문 커밋
SMTP_PASSWORD 평문 커밋
POSTGRES_PASSWORD 평문 커밋
```

허용:

```text
서버 내부 .env.production
GitHub Actions Secrets
호스팅/클라우드 Secret Store
```

운영 배포 자동화는 GitHub Actions Secrets에서 값을 읽어 서버의
`.env.production`을 생성하거나 갱신한다.
