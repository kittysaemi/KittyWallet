# 4. 동기화 구조

## 기본 원칙

미냥이 지갑은 Offline First 구조를 사용한다.

서버보다 로컬 저장이 먼저 수행된다.

---

## 처리 흐름

```text
사용자 저장
 ↓
IndexedDB 저장
 ↓
Sync Queue 등록
 ↓
동기화 대기
 ↓
Upload API 호출
 ↓
서버 저장
 ↓
Queue 제거
```

---

## 저장 실패

```text
사용자 저장
 ↓
IndexedDB 저장
 ↓
Queue 유지
 ↓
재시도
```

# 5. Sync Queue 정책

## Queue 상태

| 상태       | 설명    |
| -------- | ----- |
| pending  | 대기    |
| syncing  | 동기화 중 |
| success  | 완료    |
| failed   | 실패    |
| conflict | 충돌    |

---

## Queue 삭제 조건

허용

```text
success
```

---

금지

```text
failed
conflict
```

---

## Queue 보존 기간

| 상태       | 기간       |
| -------- | -------- |
| success  | 즉시 삭제    |
| failed   | 30일      |
| conflict | 관리자 확인 전 |
