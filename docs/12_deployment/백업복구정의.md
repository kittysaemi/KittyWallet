# 1. 문서 목적

본 문서는 미냥이 지갑 서비스의 데이터 백업 및 복구 정책을 정의한다.

본 문서는 다음 목적을 가진다.

* 재해 복구 기준 수립
* 법적 및 운영 감사 대응

# 17. IndexedDB 복구 정책

## 원칙

IndexedDB는 백업 대상 아님

---

## 이유

클라이언트 저장소

---

## 복구 방법

```text
Sync Queue 재동기화
```

---

## 삭제 정책

IndexedDB 삭제는 복구 절차의 기본 조치가 아니다.

삭제는 다음 조건을 모두 만족할 때만 최후 수단으로 허용한다.

| 조건 | 기준 |
|---|---|
| 사용자 확인 | 사용자가 삭제 위험을 인지하고 명시적으로 동의 |
| Sync Queue export | 삭제 전 Queue 데이터를 export |
| pending count 확인 | pending Sync Queue가 0건임을 확인 |
| 서버 반영 확인 | 대상 거래가 PostgreSQL `TRANSACTION`에 반영되었는지 확인 |
| 운영 승인 | 개발/운영 승인 완료 |

---

## 삭제 전 복구 우선순위

```text
1. Service Worker 재등록
2. Cache Storage 초기화
3. Sync Queue 재동기화
4. 서버 반영 여부 확인
5. 최후 수단으로 IndexedDB 삭제 검토
```
