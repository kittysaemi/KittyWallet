# 서버구성정의.md

# 1. 문서 목적

본 문서는 미냥이 지갑 서비스의 서버 구성 및 인프라 구조를 정의한다.

본 문서는 다음 목적을 가진다.

* 개발 환경 표준화
* 운영 환경 구성 기준 제공
* 배포 환경 정의
* 서버 확장 기준 제공
* 신규 개발자 온보딩 지원

---

# 2. 적용 범위

본 문서는 다음 대상에 적용한다.

| 구분            | 대상             |
| ------------- | -------------- |
| Frontend      | React PWA      |
| Backend       | Node.js API    |
| Database      | PostgreSQL     |
| Reverse Proxy | Nginx          |
| Container     | Docker         |
| CI/CD         | GitHub Actions |

---

# 3. 서버 구성 원칙

## 단순 구조 우선

초기 서비스는 단일 서버 기반으로 운영한다.

```text
1 Server
1 Database
1 Reverse Proxy
```

---

## Docker 기반

모든 서비스는 Docker Container로 운영한다.

---

## 환경 분리

개발 환경과 운영 환경을 분리한다.

---

## Stateless 원칙

API 서버는 상태를 저장하지 않는다.

모든 데이터는 PostgreSQL에 저장한다.
