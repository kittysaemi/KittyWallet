# 장애대응절차.md

# 1. 문서 목적

본 문서는 미냥이 지갑 서비스 운영 중 발생하는 장애에 대한 표준 대응 절차를 정의한다.

본 문서는 다음 목적을 가진다.

* 장애 대응 시간 단축
* 데이터 손실 방지
* 운영 절차 표준화
* 롤백 기준 제공
* Hotfix 적용 기준 제공

# 3. 장애 등급 정의

## P1 (Critical)

서비스 핵심 기능 사용 불가

예시

* 로그인 불가
* API 전체 장애
* DB 장애
* 데이터 손실
* Sync 전체 장애

목표 대응 시간

```text
15분 이내
```

---

## P2 (Major)

일부 핵심 기능 장애

예시

* 거래 등록 실패
* 통계 오류
* 카드 등록 실패

목표 대응 시간

```text
1시간 이내
```

---

## P3 (Minor)

일부 기능 또는 UI 오류

예시

* 화면 깨짐
* 잘못된 문구
* 아이콘 표시 오류

목표 대응 시간

```text
다음 배포 반영
```

# 6. 장애 유형별 대응

## AUTH 장애

증상

* 로그인 실패
* 토큰 재발급 실패

확인

```http
POST /api/v1/auth/login
POST /api/v1/auth/refresh
```

조치

* JWT Secret 확인
* Redis(Session) 확인
* DB 연결 확인

---

## API 장애

확인

```bash
docker logs kittywallet-api
```

조치

* API 재시작
* DB 연결 확인
* 최근 배포 확인

---

## DB 장애

확인

```sql
SELECT NOW();
```

조치

* Connection Pool 확인
* Disk 확인
* 백업 상태 확인

---

## Sync 장애

확인

```sql
SELECT count(*)
FROM sync_queue
WHERE status='FAILED';
```

조치

* Upload API 확인
* Queue 상태 확인
* 재동기화 수행

---

## PWA 장애

확인

Chrome DevTools

```text
Application
 └ Service Workers
```

조치

* Cache Storage 초기화
* Service Worker 재배포
* IndexedDB 및 Sync Queue는 삭제하지 않음

---

## PWA 데이터 저장소 장애

증상

* IndexedDB 읽기/쓰기 실패
* Sync Queue 처리 불가
* 미동기화 거래가 남아 있는 상태에서 앱 실행 불안정

확인

```text
Application
 └ IndexedDB
     └ Sync Queue pending count
```

조치 원칙

| 조치 | 기준 |
|---|---|
| Cache Storage 삭제 | 허용. 앱 리소스 또는 조회 캐시 손상 시 수행 |
| Service Worker 재등록 | 허용. 업데이트 또는 캐시 손상 복구 시 수행 |
| IndexedDB 삭제 | 사용자 확인 후 최후 수단 |
| Sync Queue 삭제 | 금지. 삭제 전 export 및 서버 반영 확인 필수 |

IndexedDB 삭제 전 필수 절차

```text
1. 사용자에게 미동기화 데이터 유실 가능성 안내
2. Sync Queue export
3. pending Sync Queue count 확인
4. 서버 TRANSACTION 반영 여부 확인
5. 개발/운영 승인 기록
6. 사용자 최종 확인 후 IndexedDB 삭제
```
