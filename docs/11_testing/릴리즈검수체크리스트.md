# 릴리즈 검수 체크리스트

## 1. 문서 목적

본 문서는 미냥이 지갑의 개발 완료 후 배포 가능 여부를 판단하기 위한 최종 검수 체크리스트를 정의한다.

릴리즈 검수는 기능 정상 동작뿐 아니라 데이터 정합성, 오류 처리, 모바일/PWA 환경, 오프라인 동기화, 회귀 테스트 결과까지 확인한 뒤 배포 여부를 결정하는 마지막 기준이다.

---

## 2. 릴리즈 판정 기준

| 판정 | 기준 |
|---|---|
| 배포 가능 | Critical/High 항목 모두 PASS, 치명적 오류 없음 |
| 조건부 가능 | Medium 이하 오류만 존재하고 우회 방법 있음 |
| 배포 보류 | Critical 또는 High 오류 존재 |
| 배포 불가 | 인증, 거래, 잔액, 통계, 동기화 오류 존재 |

---

## 3. 검수 상태값

| 상태 | 의미 |
|---|---|
| PASS | 정상 |
| FAIL | 실패 |
| N/A | 해당 없음 |
| BLOCKED | 선행 오류로 검수 불가 |

# 20. 배포 보류 기준

다음 중 하나라도 해당하면 배포를 보류한다.

- 로그인 불가
- 거래 등록/수정/삭제 불가
- 계좌 잔액 계산 오류
- 삭제 거래가 통계에 포함됨
- 다른 사용자의 데이터가 노출됨
- 오프라인 저장 데이터가 유실됨
- 동기화 중 중복 거래 생성
- API 공통 응답 구조 불일치
- 모바일 화면에서 주요 기능 사용 불가
- Critical 또는 High 오류 미해결
