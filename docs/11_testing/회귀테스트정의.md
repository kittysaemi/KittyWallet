# 회귀 테스트 정의

## 1. 문서 목적

본 문서는 미냥이 지갑에서 기능 수정, 오류 수정, API 변경, UI 변경, 정책 변경, 배포 전 검수 시 기존 기능이 깨지지 않았는지 확인하기 위한 회귀 테스트 기준을 정의한다.

회귀 테스트는 새 기능 검증보다 더 중요할 수 있다. 특히 거래, 잔액, 통계, 동기화처럼 서로 연결된 기능은 한 영역 수정이 다른 영역 장애로 이어질 수 있으므로 영향 범위를 명확히 관리한다.

# 6. 회귀 테스트 실행 절차

1. 변경된 기능 또는 파일을 확인한다.
2. 변경 영역별 회귀 테스트 매트릭스를 확인한다.
3. R1/R2/R3 중 실행 범위를 선택한다.
4. 관련 테스트 케이스와 E2E 시나리오를 실행한다.
5. 실패 항목을 이슈로 등록한다.
6. 수정 후 동일 항목을 재실행한다.
7. 모든 필수 항목이 PASS일 때 배포 가능으로 판단한다.

---

# 7. AI 회귀 테스트 수행 기준

AI가 회귀 테스트를 수행할 때는 다음 순서로 처리한다.

1. 변경 파일 목록 분석
2. 영향받는 기능 영역 추론
3. 본 문서의 변경 영역별 매트릭스와 매칭
4. 실행할 테스트 케이스 목록 생성
5. 자동화 가능한 항목은 테스트 코드 생성 또는 실행
6. 수동 검수가 필요한 항목은 체크리스트로 분리
7. 실패 결과와 수정 후보 파일 정리
