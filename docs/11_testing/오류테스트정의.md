# 오류 테스트 정의

## 1. 문서 목적

본 문서는 미냥이 지갑에서 발생 가능한 오류, 예외 상황, 실패 응답, 네트워크 문제, 동기화 실패 상황을 검증하기 위한 기준을 정의한다.

오류 테스트는 정상 기능보다 더 중요하게 관리해야 한다. 특히 거래 저장, 잔액 계산, 삭제 거래 제외, 오프라인 동기화는 오류 발생 시 데이터 불일치가 생길 수 있으므로 반드시 검증한다.

---

## 2. 오류 테스트 범위

| 구분 | 검증 대상 |
|---|---|
| Validation 오류 | 필수값 누락, 형식 오류, 범위 오류 |
| 인증 오류 | 로그인 실패, 토큰 만료, 권한 없음 |
| API 오류 | 400, 401, 403, 404, 409, 500 |
| 거래 오류 | 미래 날짜, 금액 오류, 존재하지 않는 계좌/카드 |
| 데이터 오류 | 삭제 데이터 노출, 잔액 불일치, 통계 불일치 |
| 네트워크 오류 | 오프라인, 요청 실패, timeout |
| 동기화 오류 | 충돌, 중복 전송, 재시도 실패 |
| UI 오류 | loading 고착, error 미표시, 버튼 중복 클릭 |

---

## 3. 공통 실패 응답 검증 기준

실패 응답은 다음 구조를 따른다.

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "ERROR_CODE",
    "message": "오류 메시지",
    "details": {}
  }
}
```

| 검증 항목 | 기대 결과 |
|---|---|
| success | false |
| data | null |
| error.code | 정의된 오류 코드 사용 |
| error.message | 사용자 또는 개발자가 이해 가능한 메시지 |
| error.details | 필요한 경우 상세 정보 포함 |
