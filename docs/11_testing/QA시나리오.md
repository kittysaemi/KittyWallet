# QA 시나리오

## 1. 문서 목적

본 문서는 미냥이 지갑의 기능 개발 완료 후 사용자 관점에서 서비스가 정상 동작하는지 검수하기 위한 QA 시나리오를 정의한다.

QA 시나리오는 단순 기능 확인이 아니라 실제 사용 흐름, 화면 상태, 예외 상황, 모바일/PWA 환경, 오프라인 동작까지 포함하여 검수하는 기준 문서로 사용한다.

본 문서는 AI가 QA 항목을 해석하고 테스트 자동화 코드 또는 수동 검수 체크리스트를 생성할 때 기준으로 사용한다.

---

## 2. 적용 범위

| 구분 | 적용 내용 |
|---|---|
| 인증 | 회원가입, 로그인, 로그아웃, 토큰 만료 |
| 거래 | 거래 등록, 수정, 삭제, 검색 |
| 계좌 | 계좌 등록, 수정, 사용 여부 |
| 카드 | 카드 등록, 수정, 사용 여부 |
| 카테고리 | 카테고리 등록, 수정, 표시 여부 |
| 아이콘 | 아이콘 선택, 아이콘 목록 표시 |
| 통계 | 월별/기간별/카테고리 통계 |
| 설정 | 사용자 정보, 앱 설정, 캐시 초기화 |
| PWA | 설치, 캐시, 오프라인 표시 |
| 동기화 | 오프라인 저장, Sync Queue, 온라인 복구 |

---

## 3. QA 기본 원칙

- 사용자가 실제로 사용하는 흐름 기준으로 검수한다.
- 화면별 `loading`, `empty`, `error`, `offline`, `syncing` 상태를 확인한다.
- API 성공 응답뿐 아니라 실패 응답도 검수한다.
- 모바일 브라우저 환경을 기준으로 검수한다.
- 오프라인 상태에서 가능한 기능과 불가능한 기능을 분리하여 검수한다.
- 테스트 결과는 통과/실패/보류로 기록한다.

---

## 4. QA 결과 상태

| 상태 | 의미 |
|---|---|
| PASS | 기대 결과와 동일하게 동작 |
| FAIL | 기대 결과와 다르게 동작 |
| BLOCKED | 선행 기능 오류로 검수 불가 |
| NEED_CHECK | 정책 또는 기획 확인 필요 |

# 5. 인증 QA 시나리오

## QA-AUTH-001 로그인 성공

| 항목 | 내용 |
|---|---|
| 목적 | 정상 계정으로 로그인 가능한지 확인 |
| 사전 조건 | 회원가입된 사용자 계정 존재 |
| 절차 | 1. 로그인 화면 진입<br>2. 이메일 입력<br>3. 비밀번호 입력<br>4. 로그인 버튼 클릭 |
| 기대 결과 | 로그인 성공 후 메인 대시보드로 이동한다. 사용자 상태가 초기화되고 토큰이 저장된다. |
| 확인 항목 | `/dashboard` 이동, 사용자 카드 표시, 최근 거래 조회 요청 |
| 우선순위 | High |

## QA-AUTH-002 로그인 실패

| 항목 | 내용 |
|---|---|
| 목적 | 잘못된 계정 정보 입력 시 오류가 표시되는지 확인 |
| 사전 조건 | 로그인 화면 진입 |
| 절차 | 1. 존재하지 않는 이메일 또는 틀린 비밀번호 입력<br>2. 로그인 버튼 클릭 |
| 기대 결과 | 로그인 실패 메시지가 표시되고 대시보드로 이동하지 않는다. |
| 확인 항목 | 오류 메시지, 토큰 미저장, 현재 화면 유지 |

| 확인 항목 | reset_token 검증, 비밀번호 확인 불일치 검증, 기존 세션 만료 |

| 절차 | 1. 거래 등록 화면 진입<br>2. 거래 유형에서 수입 또는 지출 선택<br>3. 금액 입력<br>4. 날짜 선택<br>5. 결제수단으로 계좌 또는 카드 선택<br>6. 카테고리 선택<br>7. 저장 클릭 |

| 확인 항목 | 거래 ID 생성, 목록 반영, 계좌 거래 시 잔액 반영, 카드 거래 시 계좌 잔액 미변경 |
