# 테스트 케이스 정의

## 1. 문서 목적

본 문서는 미냥이 지갑의 기능별 테스트 케이스를 정의한다.

테스트 케이스는 단위 테스트, 통합 테스트, API 테스트, E2E 테스트 코드 생성의 기준으로 사용한다. AI는 본 문서를 기준으로 테스트 데이터를 구성하고, 정상/실패/예외 케이스를 자동화 코드로 변환할 수 있어야 한다.

---

## 2. 테스트 케이스 작성 규칙

| 항목 | 설명 |
|---|---|
| 테스트 ID | 기능 그룹별 고유 ID 사용 |
| 테스트 유형 | Unit, Integration, API, E2E 중 선택 |
| 우선순위 | Critical, High, Medium, Low |
| 사전 조건 | 테스트 실행 전 필요한 데이터 또는 상태 |
| 입력값 | 테스트에 사용할 입력 데이터 |
| 기대 결과 | 성공/실패 기준 |
| 관련 문서 | 화면/API/정책/오류코드 문서 연결 |

| 문서 항목 | 생성 대상 |
|---|---|
| Unit | validation, 계산 함수, 상태 처리 테스트 |
| API | REST API 요청/응답 테스트 |
| Integration | DB, 상태관리, 동기화 연동 테스트 |
| E2E | Playwright 기반 사용자 흐름 테스트 |
| UI | 컴포넌트 렌더링 및 상태 표시 테스트 |
