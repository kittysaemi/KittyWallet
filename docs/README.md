# 미냥이 지갑 개발 문서

## 프로젝트 소개

미냥이 지갑은 계좌, 카드, 거래 내역을 통합 관리할 수 있는 PWA 기반 모바일 가계부 서비스이다.

본 프로젝트는 단순 화면 구현이 아닌, 유지보수성과 확장성을 고려한 구조 중심 개발을 목표로 한다.

프로젝트는 다음 목표를 기준으로 개발한다.

* 모바일 중심 UX 제공
* 빠른 거래 입력 환경 제공
* 계좌/카드/거래 통합 관리
* 오프라인 대응 가능한 PWA 구조 제공
* API 변경에 유연한 구조 설계
* 장기 유지보수가 가능한 아키텍처 구성
* AI(Codex/GPT) 협업 가능한 문서 기반 개발 환경 구성

# 개발 원칙

본 프로젝트는 다음 원칙을 기준으로 개발한다.

## 1. 정책 기반 개발

정책 정의 없이 기능을 구현하지 않는다.

반드시 정의:

* 입력 제한
* 삭제 정책
* 로그인 정책
* 동기화 정책
* 오프라인 정책
* 캐시 정책

## 2. 상태 기반 UI 개발

모든 화면은 상태를 정의해야 한다.

반드시 포함:

* loading
* empty
* error
* offline

필요 시 추가:

* refreshing
* syncing
* expired

## 3. 예외 처리 필수

Happy Path만 구현하지 않는다.

반드시 고려:

* API 실패
* 네트워크 끊김
* 저장 실패
* 토큰 만료
* 중복 요청
* 오프라인 저장 실패

## 4. API 우선 개발

화면 구현 전에 API 명세를 먼저 정의한다.

필수 포함:

* Request
* Response
* Error
* Validation
* 권한 조건

## 5. 구조 기반 개발

단순 기능 구현보다 유지보수 가능한 구조를 우선한다.

목표:

* 기능 추가 시 영향 최소화
* API 변경 시 수정 범위 최소화
* 공통 로직 재사용 가능 구조 유지

## 6. 모바일/PWA 기준 개발

모든 기능은 모바일 기준으로 설계한다.

반드시 고려:

* Safe Area
* 모바일 키보드
* 터치 영역
* 오프라인 환경
* 앱 설치 UX
* Service Worker 동작

# AI 협업 개발 규칙

Codex/GPT 기반 작업 시 아래 규칙을 따른다.

## 반드시 참조해야 하는 문서

* 정책 정의
* 상태 정의
* 예외 처리 정의
* API 명세
* 데이터 구조 정의
* ERD

## 금지 사항

* 정책 없는 기능 구현 금지
* 상태 정의 없는 UI 구현 금지
* 임의 DB 컬럼 추가 금지
* API 명세 없는 API 구현 금지
* 화면 정의 없는 UI 추가 금지

## 구현 필수 항목

* validation 처리
* loading 상태 처리
* error 상태 처리
* offline 대응 처리
* 공통 응답 규격 준수
