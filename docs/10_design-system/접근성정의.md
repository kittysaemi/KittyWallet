# 접근성정의.md

# 1. 문서 목적

본 문서는 미냥이 지갑 서비스의 접근성 기준을 정의한다.

모든 사용자가 신체 능력, 환경, 기기 차이에 관계없이 서비스를 이용할 수 있도록 접근성 규칙을 적용한다.

본 문서는 모바일 웹(PWA), 향후 하이브리드 앱 환경까지 포함한다.

---

# 2. 접근성 목표

## 사용성 확보

누구나 쉽게 사용할 수 있어야 한다.

---

## 정보 전달

색상에 의존하지 않고 정보를 전달해야 한다.

---

## 모바일 최적화

작은 화면에서도 오작동 없이 사용할 수 있어야 한다.

---

## 금융 데이터 가독성 확보

금액과 거래 정보는 가장 쉽게 읽을 수 있어야 한다.

---

# 3. 적용 범위

적용 대상

* 전체 화면
* 공통 컴포넌트
* 입력 화면
* 차트
* 모달
* Bottom Sheet
* Toast
* Offline Banner

---

# 4. 접근성 설계 원칙

## 인지 가능성

모든 정보는 사용자가 쉽게 인지할 수 있어야 한다.

---

## 조작 가능성

모든 기능은 쉽게 터치할 수 있어야 한다.

---

## 이해 가능성

UI와 문구는 직관적으로 이해 가능해야 한다.

---

## 견고성

브라우저 및 보조 기술 환경에서도 동작해야 한다.

---

# 5. 터치 영역 규칙

## 최소 터치 영역

```text
44px × 44px
```

---

적용 대상

* 버튼
* 아이콘 버튼
* 체크박스
* 스위치
* 탭 메뉴
* 하단 네비게이션

---

금지

```text
44px 미만 터치 영역
```

---

# 6. 텍스트 규칙

## 최소 폰트 크기

| 구분      | 크기      |
| ------- | ------- |
| Caption | 12px    |
| Body    | 14px    |
| Heading | 18px 이상 |

---

## 줄 간격

```text
1.4 이상
```

---

## 문장 길이

한 문장은 가능한 짧게 유지한다.

---

좋음

```text
저장되었어요.
```

---

나쁨

```text
거래 데이터가 성공적으로 저장되었습니다.
```

---

# 7. 색상 접근성 규칙

## 색상만으로 상태 표현 금지

금지

```text
빨간색만 표시
```

---

허용

```text
오류 아이콘 + 오류 문구 + 색상
```

---

## 상태 표시 규칙

| 상태  | 아이콘 | 문구 | 색상 |
| --- | --- | -- | -- |
| 성공  | O   | O  | O  |
| 오류  | O   | O  | O  |
| 경고  | O   | O  | O  |
| 동기화 | O   | O  | O  |

---

# 8. 명도 대비 규칙

일반 텍스트

```text
4.5 : 1 이상
```

---

큰 텍스트

```text
3 : 1 이상
```

---

Primary 색상 사용 시

텍스트 가독성 확인 필수

---

# 9. 아이콘 접근성 규칙

## 단독 아이콘 사용 금지

금지

```text
🗑
```

---

허용

```text
🗑 삭제
```

---

## Icon Button

반드시 aria-label 제공

예시

```html
<button aria-label="거래 삭제">
```

---

# 10. 입력 화면 접근성

## Label 필수

금지

```text
Placeholder만 사용
```

---

허용

```text
거래 금액
[ 입력 ]
```

---

## 오류 메시지

오류 발생 시 입력 필드 아래 표시

예시

```text
금액을 입력해주세요.
```

---

## 필수값 표시

```text
거래 금액 *
```

---

# 11. Form 접근성

## 포커스 이동

다음 입력 항목으로 자연스럽게 이동해야 한다.

---

## 키패드 자동 선택

금액 입력

```html
inputmode="numeric"
```

---

날짜 입력

```html
type="date"
```

---

# 12. Button 접근성

## Loading 상태

저장 중에는 중복 클릭 불가

---

표시

```text
저장 중...
```

---

## Disabled 상태

비활성 이유가 명확해야 한다.

---

# 13. Modal 접근성

## 포커스 유지

모달이 열리면 모달 내부에 포커스를 유지한다.

---

## 닫기 버튼

반드시 제공한다.

---

## 배경 영역

스크린 리더 접근 제외

---

# 14. Bottom Sheet 접근성

## 제목 제공

예시

```text
카테고리 선택
```

---

## 스크롤 지원

항목이 많을 경우 스크롤 가능해야 한다.

---

## 닫기 제공

상단 닫기 버튼 필수

---

# 15. Toast 접근성

## 자동 사라짐

3초 기본

---

## 중요 오류

Toast만 표시하지 않는다.

Error State도 함께 제공한다.

---

# 16. Empty State 접근성

## 필수 요소

| 항목    | 필수 |
| ----- | -- |
| 제목    | O  |
| 설명    | O  |
| 액션 버튼 | O  |

---

예시

```text
아직 거래 내역이 없어요.

첫 거래를 기록해보세요.

[거래 등록]
```

---

# 17. Offline Banner 접근성

## 필수 문구

```text
현재 오프라인 상태예요.
```

---

## 보조 문구

```text
저장한 내용은 연결되면 자동으로 동기화돼요.
```

---

아이콘 포함

```text
⚠ 오프라인 상태
```

---

# 18. Transaction Card 접근성

## 금액 강조

가장 먼저 인식 가능해야 한다.

---

표시 우선순위

```text
금액
거래명
날짜
메모
```

---

## 수입 / 지출

색상 외에도 표시

```text
+ 50,000원
```

```text
- 15,000원
```

---

# 19. 차트 접근성

반드시 수치 제공

---

허용


---



# 23. 금지 규칙

금지

* Placeholder만 사용하는 입력창
* 색상만으로 상태 표시
* 설명 없는 오류 메시지
* 44px 미만 터치 영역
* 자동 사라지는 중요 메시지
* 차트만 표시하고 수치 미제공

---

# 24. 미냥이 지갑 접근성 원칙

미냥이 지갑의 접근성 목표는

```text
누구나 쉽게 입력할 수 있어야 한다.
누구나 쉽게 상태를 이해할 수 있어야 한다.
누구나 쉽게 데이터를 읽을 수 있어야 한다.
```

귀여움보다 사용성을 우선한다.
