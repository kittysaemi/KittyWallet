# 디자인시스템정의.md

# 1. 문서 목적

본 문서는 미냥이 지갑 서비스의 UI 디자인 시스템을 정의한다.

디자인 시스템은 화면 구현 시 일관된 사용자 경험을 제공하고, 공통 컴포넌트 재사용성을 높이며, 프론트엔드 개발과 디자인 작업의 기준 문서로 사용한다.

모든 화면 및 컴포넌트는 본 문서를 기준으로 설계 및 구현한다.

---

# 2. 적용 범위

적용 대상

* 모바일 웹(PWA)
* 향후 Capacitor 기반 하이브리드 앱
* 관리자 기능(추후 확장 시)
* 공통 컴포넌트
* 화면 레이아웃
* 상태 UI

적용 제외

* 마케팅 페이지
* 외부 연동 페이지

---

# 3. 디자인 시스템 목표

## 일관성

동일한 기능은 동일한 UI 패턴을 사용한다.

## 단순성

복잡한 UI보다 빠른 입력과 쉬운 사용을 우선한다.

## 재사용성

모든 UI는 공통 컴포넌트 기반으로 구성한다.

## 모바일 최적화

모바일 환경을 최우선으로 설계한다.

## 확장성

신규 기능 추가 시 기존 UI 구조를 유지할 수 있어야 한다.

---

# 4. 디자인 원칙

## Mobile First

모든 화면은 모바일 화면을 기준으로 설계한다.

## Fast Input

거래 등록은 최소 입력 단계로 구성한다.

## Consistency

동일 기능은 동일한 형태의 UI를 사용한다.

## Accessibility

모든 사용자가 접근 가능한 UI를 제공한다.

## Offline Aware

오프라인 상태를 사용자가 명확하게 인지할 수 있어야 한다.

# 18. 공통 UI 개발 규칙

모든 화면은 공통 컴포넌트를 사용한다.

개별 화면에서 독자적인 Button, Input, Modal 생성 금지.

공통 UI 수정 시 전체 화면에 동일하게 반영되어야 한다.
