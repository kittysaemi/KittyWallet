# UX라이팅가이드.md

# 1. 문서 목적

본 문서는 미냥이 지갑 서비스에서 사용자에게 노출되는 모든 문구 작성 기준을 정의한다.

UX 라이팅은 단순 텍스트가 아니라 사용자의 행동을 유도하고 서비스 상태를 전달하는 UI 구성 요소의 일부로 간주한다.

모든 화면, 컴포넌트, 오류 메시지, 상태 메시지는 본 문서를 기준으로 작성한다.

---

# 2. UX 라이팅 목표

## 쉽고 빠른 이해

사용자가 문장을 읽는 시간을 최소화한다.

---

## 친근함

부드럽고 부담 없는 표현을 사용한다.

---

## 신뢰성

금융 데이터 서비스 특성상 과도한 캐릭터 표현을 사용하지 않는다.

---

## 행동 유도

현재 상태와 다음 행동을 명확하게 안내한다.

---

## 상태 전달

Loading, Error, Offline, Syncing 상태를 사용자가 쉽게 이해할 수 있어야 한다.

---

# 18. 가장 중요한 UX 원칙

미냥이 지갑은 귀여운 앱이 아니라

```text
입력하기 쉬운 앱
상태가 잘 보이는 앱
안심하고 사용하는 앱
```

이어야 한다.

모든 문구는 귀여움보다 이해하기 쉬움을 우선한다.
