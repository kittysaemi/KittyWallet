# 컴포넌트정의.md

# 1. 문서 목적

본 문서는 미냥이 지갑 서비스에서 사용하는 공통 UI 컴포넌트의 구조, 사용 규칙, 상태, 디자인 기준을 정의한다.

모든 화면은 본 문서의 컴포넌트를 기준으로 구성하며, 화면별로 독립적인 UI 컴포넌트를 중복 생성하지 않는다.

---

# 2. 컴포넌트 설계 목표

## 재사용성

동일한 UI는 하나의 공통 컴포넌트로 관리한다.

## 일관성

버튼, 입력창, 카드, 모달 등 주요 UI는 동일한 규칙을 따른다.

## 모바일 최적화

모든 컴포넌트는 모바일 터치 환경을 기준으로 설계한다.

## 고양이 테마 반영

둥근 형태, 부드러운 색상, 귀여운 강조 요소를 사용하되 가독성을 해치지 않는다.

## 상태 기반 UI

Loading, Error, Empty, Offline, Syncing 상태를 명확하게 표현한다.

---

# 3. 컴포넌트 공통 규칙

## 기본 단위

8px 기준으로 여백을 구성한다.

## 최소 터치 영역

44px 이상을 유지한다.

## Radius

기본적으로 둥근 모서리를 사용한다.

| 컴포넌트         | Radius        |
| ------------ | ------------- |
| Button       | 12px          |
| Input        | 12px          |
| Card         | 16px          |
| Modal        | 20px          |
| Bottom Sheet | 24px 24px 0 0 |

## 그림자

카드와 모달은 부드러운 핑크 계열 그림자를 사용한다.

```css
box-shadow: 0 4px 16px rgba(253, 165, 227, 0.15);
```

# 4. Button 컴포넌트

## 목적

사용자의 주요 액션을 실행하는 컴포넌트이다.

## 종류

| 종류               | 용도                  |
| ---------------- | ------------------- |
| Primary Button   | 저장, 등록, 로그인 등 주요 액션 |
| Secondary Button | 보조 액션               |
| Ghost Button     | 가벼운 이동, 취소          |
| Danger Button    | 삭제, 위험 작업           |
| Icon Button      | 아이콘 기반 액션           |

---

## Primary Button

```text
배경: #FDA5E3
텍스트: #3D2E38
Radius: 12px
높이: 44px 이상
```

사용 예

* 로그인
* 거래 저장
* 계좌 등록
* 카드 등록

---

## Secondary Button

```text
배경: #FFF4FB
Border: #F4C8E4
텍스트: #6F5B67
```

사용 예

* 취소
* 이전
* 보조 이동

---

## Danger Button

```text
배경: #F28A9B
텍스트: #FFFFFF
```

사용 예

* 거래 삭제
* 로그아웃 확인
* 위험 작업

---

## Button 상태

| 상태       | 처리                           |
| -------- | ---------------------------- |
| Default  | 기본 표시                        |
| Pressed  | 살짝 어두운 색상 적용                 |
| Disabled | 클릭 불가, 투명도 50%               |
| Loading  | 버튼 내부 Spinner 표시             |
| Offline  | 저장 가능 여부에 따라 비활성 또는 임시 저장 표시 |

---

# 5. Input 컴포넌트

## 목적

사용자의 텍스트, 숫자, 날짜 입력을 처리한다.

## 종류

| 종류             | 용도      |
| -------------- | ------- |
| Text Input     | 일반 텍스트  |
| Number Input   | 금액 입력   |
| Date Input     | 거래 날짜   |
| Search Input   | 거래 검색   |
| Password Input | 비밀번호 입력 |

---

## 기본 스타일

```text
높이: 44px 이상
배경: #FFFFFF
Border: #F4C8E4
Radius: 12px
텍스트: #3D2E38
Placeholder: #9B8792
```

---

## 상태

| 상태       | 처리                        |
| -------- | ------------------------- |
| Default  | 기본 입력 상태                  |
| Focus    | Primary Border 적용         |
| Error    | Danger Border 및 오류 메시지 표시 |
| Disabled | 배경 비활성 색상 적용              |
| Readonly | 값은 보이지만 수정 불가             |

---

## 금액 입력 규칙

* 숫자 키패드 사용
* 천 단위 콤마 표시
* 0 이하 금액 저장 불가
* 입력 중에도 가독성 유지

---

# 6. Select 컴포넌트

## 목적

계좌, 카드, 카테고리, 아이콘 등을 선택한다.

## 종류

| 종류              | 용도      |
| --------------- | ------- |
| Account Select  | 계좌 선택   |
| Card Select     | 카드 선택   |
| Category Select | 카테고리 선택 |
| Icon Select     | 아이콘 선택  |
| Period Select   | 기간 선택   |

---

## 기본 규칙

* 선택 가능한 항목만 표시한다.
* 사용 여부가 OFF인 계좌, 카드, 카테고리는 선택 목록에서 제외한다.
* 모바일에서는 Bottom Sheet 선택 UI를 우선 사용한다.

---

# 7. Card 컴포넌트

## 목적

자산, 거래, 통계, 사용자 정보를 묶어서 표시한다.

## 종류

| 종류               | 용도            |
| ---------------- | ------------- |
| User Card        | 사용자 닉네임, 로그아웃 |
| Asset Card       | 자산 요약         |
| Transaction Card | 최근 거래         |
| Statistic Card   | 소비 통계         |
| Account Card     | 계좌 정보         |
| Card Item        | 카드 정보         |
| Category Card    | 카테고리 정보       |

---

## 기본 스타일

```text
배경: #FFFFFF
Radius: 16px
Padding: 16px
Shadow: soft pink shadow
Border: #F8DDF0
```

---

## 고양이 테마 표현

* 카드 상단에 작은 포인트 아이콘 사용 가능
* 선택된 카드는 발바닥 느낌의 Primary Soft 배경 사용
* 과한 캐릭터 장식은 데이터 가독성을 해치므로 금지

---

# 8. Transaction Card 컴포넌트

## 목적

거래 내역을 목록 형태로 표시한다.

## 구성 요소

| 요소       | 설명            |
| -------- | ------------- |
| 카테고리 아이콘 | 거래 카테고리 표시    |
| 거래명/메모   | 거래 내용 표시      |
| 날짜       | 거래 날짜         |
| 금액       | 수입/지출 금액      |
| 동기화 상태   | 오프라인 저장 여부 표시 |

---

## 금액 색상

| 유형 | 색상      |
| -- | ------- |
| 수입 | Success |
| 지출 | Danger  |
| 이체 | Info    |

---

## 상태 표시

| 상태      | 표시                         |
| ------- | -------------------------- |
| syncing | 작은 동기화 아이콘                 |
| failed  | 재시도 버튼                     |
| offline | 임시 저장 배지                   |
| deleted | 목록에서 즉시 제거하지 않고 삭제 처리 중 표시 |

---

# 9. Bottom Sheet 컴포넌트

## 목적

모바일 환경에서 선택, 상세 보기, 확인 액션을 제공한다.

## 사용 위치

* 계좌 선택
* 카드 선택
* 카테고리 선택
* 아이콘 선택
* 거래 상세
* 필터 설정

---

## 기본 스타일

```text
배경: #FFFFFF
상단 Radius: 24px
Padding: 20px
최대 높이: 화면의 80%
```

---

## 규칙

* 닫기 버튼 제공
* 배경 클릭 종료는 선택 적용
* 스크롤 가능해야 함
* 하단 Safe Area를 고려해야 함

---

# 10. Modal 컴포넌트

## 목적

사용자의 확인이 필요한 작업을 처리한다.

## 종류

| 종류                   | 용도    |
| -------------------- | ----- |
| Alert Modal          | 안내    |
| Confirm Modal        | 확인/취소 |
| Error Modal          | 오류 표시 |
| Delete Confirm Modal | 삭제 확인 |

---

## 규칙

* 제목, 본문, 버튼 영역을 구분한다.
* 위험 작업은 Danger Button을 사용한다.
* 단순 선택 UI는 Modal보다 Bottom Sheet를 우선 사용한다.

---

# 11. Toast 컴포넌트

## 목적

짧은 결과 메시지를 표시한다.

## 종류

| 종류            | 용도      |
| ------------- | ------- |
| Success Toast | 저장 완료   |
| Error Toast   | 저장 실패   |
| Warning Toast | 오프라인 안내 |
| Info Toast    | 동기화 안내  |

---

## 표시 규칙

* 화면 하단에 표시한다.
* 2~3초 후 자동 사라진다.
* 중요한 오류는 Toast만 사용하지 않고 화면 상태로도 표시한다.

---

# 12. Badge 컴포넌트

## 목적

상태나 유형을 짧게 표시한다.

## 종류

| 종류             | 용도   |
| -------------- | ---- |
| Income Badge   | 수입   |
| Expense Badge  | 지출   |
| Offline Badge  | 오프라인 |
| Sync Badge     | 동기화  |
| Disabled Badge | 비활성  |

---

## 스타일

```text
Radius: full
Padding: 4px 8px
Font Size: 12px
```

---

# 13. Icon 컴포넌트

## 목적

계좌, 카드, 카테고리, 액션을 시각적으로 표현한다.

## 크기

| 크기 | 값    |
| -- | ---- |
| XS | 16px |
| SM | 20px |
| MD | 24px |
| LG | 32px |
| XL | 48px |

---

## 규칙

* SVG 기반 아이콘 사용
* 계좌, 카드, 카테고리 아이콘은 공통 Icon 컴포넌트 사용
* 선택된 아이콘은 Primary Soft 배경 적용
* 비활성 아이콘은 Gray 계열 적용

---

# 14. Empty State 컴포넌트

## 목적

데이터가 없는 상태를 표시한다.

## 사용 위치

* 거래 내역 없음
* 검색 결과 없음
* 계좌 없음
* 카드 없음
* 통계 데이터 없음

---

## 구성 요소

| 요소    | 설명                      |
| ----- | ----------------------- |
| 아이콘   | 고양이 또는 발바닥 형태의 빈 상태 아이콘 |
| 제목    | 상태 요약                   |
| 설명    | 다음 행동 안내                |
| 액션 버튼 | 등록 또는 이동 버튼             |

---

## 예시 문구

```text
아직 거래 내역이 없어요.
첫 거래를 기록해볼까요?
```

---

# 15. Loading 컴포넌트

## 목적

데이터 조회 또는 저장 진행 중 상태를 표시한다.

## 종류

| 종류           | 용도    |
| ------------ | ----- |
| Spinner      | 버튼 내부 |
| Skeleton     | 목록 로딩 |
| Page Loading | 초기 화면 |
| Sync Loading | 동기화 중 |

---

## 규칙

* 전체 화면 로딩은 최소화한다.
* 목록은 Skeleton UI를 우선 사용한다.
* 저장 버튼은 Loading 상태에서 중복 클릭을 막는다.

---

# 16. Error State 컴포넌트

## 목적

오류 발생 상태를 표시한다.

## 사용 위치

* API 요청 실패
* 네트워크 오류
* 저장 실패
* 인증 만료
* 동기화 실패

---

## 구성 요소

| 요소     | 설명          |
| ------ | ----------- |
| 오류 아이콘 | 오류 상태 표시    |
| 오류 제목  | 사용자 친화적 메시지 |
| 상세 설명  | 원인 또는 안내    |
| 재시도 버튼 | 가능한 경우 제공   |

---

# 17. Offline Banner 컴포넌트

## 목적

네트워크 미연결 상태를 사용자에게 안내한다.

## 표시 위치

화면 상단 또는 하단 고정 영역

## 스타일

```text
배경: #FFF8E8
텍스트: #6F5B67
아이콘: Warning
```

---

## 문구 예시

```text
현재 오프라인 상태예요. 저장한 내용은 연결 후 동기화됩니다.
```

---

# 18. Sync Indicator 컴포넌트

## 목적

오프라인 저장 데이터의 동기화 상태를 표시한다.

## 상태

| 상태      | 표시     |
| ------- | ------ |
| pending | 동기화 대기 |
| syncing | 동기화 중  |
| success | 동기화 완료 |
| failed  | 동기화 실패 |

---

# 19. Header 컴포넌트

## 목적

화면 제목과 주요 액션을 표시한다.

## 구성 요소

| 요소            | 설명           |
| ------------- | ------------ |
| Back Button   | 이전 화면 이동     |
| Title         | 화면 제목        |
| Action Button | 저장, 추가, 설정 등 |

---

## 규칙

* 높이는 56px 기준으로 한다.
* Safe Area Top을 고려한다.
* 제목은 한 줄 표시를 기본으로 한다.

---

# 26. 컴포넌트 작성 규칙

## Props 규칙

* variant로 종류를 구분한다.
* size로 크기를 구분한다.
* disabled, loading, error 상태를 명시적으로 받는다.
* className 직접 주입은 최소화한다.

---

## 금지 규칙

* 화면별 Button 별도 생성 금지
